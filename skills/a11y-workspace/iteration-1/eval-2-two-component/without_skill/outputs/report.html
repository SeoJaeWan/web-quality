Navbar.tsx 검토 결과 (10건)

#심각도KWCAG 검사항목위반 내용라인권장 수정
1 중요 7.1.1 적절한 대체 텍스트 제공 햄버거 메뉴의 SVG 아이콘에 role="img"와 aria-hidden="true" 또는 <title> 요소가 없음. 스크린리더가 SVG 내부 <path>를 읽으려 시도할 수 있음. 65 SVG에 aria-hidden="true" 추가 (버튼의 aria-label이 이미 존재하므로 장식용으로 처리)
2 중요 7.2.1 키보드 접근성 햄버거 메뉴 버튼에 aria-expanded 속성이 없어 메뉴의 열림/닫힘 상태를 보조기술이 인식할 수 없음. 59-63 aria-expanded={menuOpen} 속성 추가
3 중요 7.2.1 키보드 접근성 햄버거 메뉴 aria-label이 "메뉴 열기"로 고정됨. 메뉴가 열린 상태에서도 동일한 레이블을 사용하여 현재 상태를 정확히 전달하지 못함. 63 aria-label={menuOpen ? '메뉴 닫기' : '메뉴 열기'}로 동적 변경
4 보통 7.2.4 반복 영역 건너뛰기 <nav> 요소에 aria-label이 없음. 페이지에 여러 nav 랜드마크가 있을 경우 구분 불가. 23 <nav aria-label="주 메뉴"> 추가
5 보통 7.2.4 반복 영역 건너뛰기 브레드크럼 영역이 <div>로 마크업됨. 보조기술이 이 영역을 네비게이션 랜드마크로 인식할 수 없음. 132 <nav aria-label="브레드크럼">으로 변경하고, 내부를 <ol> 리스트로 마크업
6 보통 7.4.2 제목 제공 현재 페이지를 나타내는 네비게이션 링크에 aria-current="page" 속성이 없음. 시각적으로만 파란색으로 구분됨. 37-45 활성 링크에 aria-current="page" 속성 추가
7 경미 7.3.2 예측 가능한 기능 모바일 메뉴가 열렸을 때 Escape 키로 닫을 수 없음. 키보드 사용자의 예측 가능한 동작이 보장되지 않음. 87-109 useEffect로 Escape 키 이벤트 리스너 추가하여 메뉴 닫기 처리
8 경미 7.2.1 키보드 접근성 모바일 메뉴 열림 시 포커스가 메뉴로 이동하지 않고, 닫힘 시 햄버거 버튼으로 포커스가 복귀하지 않음 (포커스 트래핑 부재). 87-109 메뉴 열림 시 첫 번째 링크로 포커스 이동, 닫힘 시 트리거 버튼으로 복귀
9 경미 7.2.1 키보드 접근성 모바일 메뉴 컨테이너에 role 및 관련 ARIA 속성이 없음. 햄버거 버튼과 메뉴 패널 간 연결이 명시적이지 않음. 88 버튼에 aria-controls="mobile-menu", 패널에 id="mobile-menu" 추가
10 경미 7.1.3 색에 무관한 인식 활성 네비게이션 링크가 색상(파란색)으로만 구분됨. 색각 이상 사용자가 현재 페이지를 식별하기 어려울 수 있음. 38-39 밑줄, font-weight 변경, 아이콘 등 추가적인 시각적 구분 제공

StatsCard.tsx 검토 결과 (5건)

#심각도KWCAG 검사항목위반 내용라인권장 수정
11 중요 7.2.1 키보드 접근성 <div>에 onClick 핸들러가 있으나 role, tabIndex, onKeyDown 속성이 없음. 키보드 사용자가 이 요소와 상호작용할 수 없음. 17 role="button" tabIndex={0} onKeyDown={handleKeyDown} 추가 또는 <button> 요소로 변경
12 중요 7.3.4 일관된 식별 id="stat-card"가 하드코딩됨. 리스트에서 여러 번 렌더링 시 동일 id가 중복되어 HTML 유효성 위반 및 보조기술 참조 오류 발생. 13 id={`stat-card-${testId}`} 등 고유한 id 생성 또는 id 속성 제거
13 보통 7.1.1 적절한 대체 텍스트 제공 클릭 가능한 요소이나 접근 가능한 이름(accessible name)이 없음. 스크린리더 사용자가 이 요소의 목적을 알 수 없음. 10-21 aria-label={label} 또는 aria-labelledby로 접근 가능한 이름 제공
14 보통 7.4.1 표의 구성 통계 수치(value)와 레이블(label)의 관계가 의미적으로 연결되지 않음. 스크린리더 사용자에게 값과 레이블의 관계가 불명확할 수 있음. 19-20 aria-labelledby 사용 또는 <dl>/<dt>/<dd> 정의형 리스트 마크업으로 변경
15 경미 7.4.2 제목 제공 값이 레이블보다 먼저 마크업에 나와 스크린리더 읽기 순서가 "값 → 레이블"이 됨. 맥락 없이 숫자가 먼저 읽힘. 19-20 DOM에서 레이블을 먼저 배치하고 CSS order로 시각적 순서 조정, 또는 aria-label로 전체 설명 제공

종합 의견

Navbar.tsx

전반적인 HTML 시맨틱 구조(<nav>, <Link>, <button>)는 양호하나, 모바일 햄버거 메뉴의 상태 관리와 관련된 ARIA 속성이 부족합니다. 특히 aria-expanded 미사용은 스크린리더 사용자가 메뉴 상태를 파악할 수 없게 만드는 주요 접근성 장벽입니다. 브레드크럼 영역의 시맨틱 마크업 개선과, 활성 링크의 aria-current 속성 추가가 필요합니다.

StatsCard.tsx

가장 심각한 문제는 <div onClick> 패턴입니다. 키보드 접근이 완전히 차단되어 있으며, KWCAG 2.2의 핵심 원칙인 "운용의 용이성"을 위반합니다. 근본적으로 <button> 요소로 변경하는 것이 최선이며, 중복 id 문제도 시급히 수정해야 합니다.

우선순위 권장 사항

  1. [즉시] StatsCard의 <div onClick>를 <button>으로 변경
  2. [즉시] StatsCard의 중복 id 제거 또는 고유값 사용
  3. [즉시] 햄버거 메뉴에 aria-expanded 추가
  4. [단기] 브레드크럼을 <nav> + <ol>로 변경
  5. [단기] 활성 링크에 aria-current="page" 추가
  6. [중기] 모바일 메뉴 포커스 관리 및 Escape 키 지원

본 리포트는 정적 코드 분석 기반으로 작성되었습니다. 실제 렌더링 환경에서의 보조기술 테스트를 추가로 권장합니다.
