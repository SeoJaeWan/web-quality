불합격 (❌)

항목 1. 적절한 대체 텍스트 — page.tsx:83

문제: <img src="/promo-banner.png">에 alt 속성이 완전히 누락되어 스크린리더가 이미지 정보를 전달할 수 없음

수정: <img src="/promo-banner.png" alt="프로모션 배너" className="mt-6 w-full" />

항목 10. 키보드 사용 보장 — StatsCard.tsx:17

문제: <div onClick={handleClick}>가 키보드 이벤트 핸들러(onKeyDown) 없이 사용됨. tabIndex도 없어 Tab 키로 포커스 불가

수정: <button onClick={handleClick} className="...">로 변경하거나, <div role="button" tabIndex={0} onClick={handleClick} onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}>

항목 17. 반복 영역 건너뛰기 — Navbar.tsx, page.tsx

문제: 페이지에 스킵 내비게이션 링크가 없음. 키보드 사용자가 매번 내비게이션 영역을 탭으로 지나가야 함

수정: 레이아웃 최상단에 <a href="#main-content" className="sr-only focus:not-sr-only">본문 바로가기</a> 추가, 본문을 <main id="main-content">로 감싸기

항목 23. 레이블과 네임 — page.tsx:59-64

문제: SVG 아이콘만 포함된 버튼 2개(btn-refresh, btn-settings)에 aria-label이 없어 스크린리더에서 "버튼"으로만 읽힘

수정: <button aria-label="새로고침" data-testid="btn-refresh">, <button aria-label="설정" data-testid="btn-settings">

항목 32. 마크업 오류 방지 — StatsCard.tsx:14, page.tsx:73

문제: (1) StatsCard에 id="stat-card" 하드코딩 — 3개 인스턴스 렌더링 시 동일 ID 중복 위반. (2) recentItems.map()에서 key prop 누락

수정: (1) id={`stat-card-${testId}`}로 고유 ID 생성. (2) {recentItems.map((item, idx) => <div key={idx}>{item}</div>)}

권고 (⚠️)

항목 8. 명도 대비

내용: page.tsx에서 text-zinc-400, text-zinc-500 색상이 흰색 배경에서 WCAG AA 기준(4.5:1) 미달 가능성이 있음. Tailwind의 zinc-400(#a1a1aa)은 약 3.5:1, zinc-500(#71717a)은 약 4.7:1. 보조 텍스트에도 최소 text-zinc-500 이상 사용 권고

항목 11. 초점 이동과 표시

내용: StatsCard의 <div onClick>에 tabIndex 없어 포커스 자체 불가. 인터랙티브 요소로 변경 후 커스텀 포커스 스타일 추가 권고

항목 33. 웹 애플리케이션 접근성

내용: Navbar.tsx 햄버거 메뉴 버튼에 aria-expanded={menuOpen} 추가하여 메뉴 열림/닫힘 상태를 보조기술에 전달해야 함

시맨틱 HTML: 랜드마크 요소

내용: page.tsx 최상위 <div>를 <main>으로 변경 권고. 브레드크럼 영역에 <nav aria-label="브레드크럼"> 사용 권고

시맨틱 HTML: 목록 구조

내용: recentItems 반복 렌더링과 네비게이션 링크를 <ul><li> 구조로 변경 권고

시맨틱 HTML: 기타 시맨틱 요소

내용: 날짜 표시에 <time> 요소 사용 권고
