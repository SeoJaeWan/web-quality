Web Performance (WP-01 ~ WP-18)

WP-01 Render-Blocking CSS High index.html 10 FAIL External stylesheet <link rel="stylesheet" href="...styles.css"> is render-blocking. Use media query hint, preload with onload swap, or async loading strategy.
WP-02 Font Display Swap Medium index.html 13-17 FAIL @font-face for "CustomFont" is missing font-display: swap. This causes Flash of Invisible Text (FOIT) during font loading, blocking LCP.
WP-03 Image Dimensions (CLS) — index High index.html 33 FAIL <img src="/public/promo.png"> is missing width and height attributes, causing Cumulative Layout Shift (CLS) as the image loads.
WP-04 Image Dimensions (CLS) — dashboard Medium dashboard.tsx 50 FAIL Dashboard promo image also lacks explicit width/height attributes. Browser cannot reserve space before the image loads.
WP-05 Lazy Loading Images Medium dashboard.tsx 50 FAIL Below-fold image lacks loading="lazy" attribute. Non-critical images should be lazily loaded to reduce initial page weight.
WP-06 Layout Thrashing High dashboard.tsx 12-13 FAIL Reading offsetHeight then immediately writing style.minHeight causes forced synchronous reflow (layout thrashing). Batch reads/writes or use requestAnimationFrame.
WP-07 CLS from Dynamic Content High dashboard.tsx 17-19 FAIL setTimeout inserts content without pre-allocated space, causing layout shift. Reserve space with min-height or use a skeleton/placeholder UI.
WP-08 Inline Script Blocking Medium index.html 37-44 WARN Inline <script> in body is parser-blocking. Consider moving to an external file with defer attribute or placing at end of body.
WP-09 Console.log in Production Low Card.tsx 13 FAIL console.log("Card clicked:", title) left in production code. Minor performance overhead and leaks internal implementation details.
WP-10 DOM Querying in Handlers Medium Card.tsx 17 WARN document.querySelector('.card-highlight') in onClick handler bypasses React's virtual DOM. Use useRef for better performance, type safety, and correctness.
WP-11 Duplicate DOM IDs Medium Card.tsx 27 FAIL id="card-item" is hardcoded. When Card renders in a list, all instances share the same ID — violating HTML spec and breaking ID-based queries.
WP-12 React List Keys Medium dashboard.tsx 45-46 FAIL List items rendered via items.map() lack a key prop. React cannot efficiently reconcile the list, causing unnecessary re-renders and potential UI bugs.
WP-13 Resource Hints (preconnect) Medium index.html - FAIL No <link rel="preconnect"> for cdn.example.com. Adding preconnect would reduce DNS + TCP + TLS latency for the external stylesheet.
WP-14 Critical CSS / Above-the-fold Medium index.html - WARN No inline critical CSS. All styles depend on the external render-blocking stylesheet. Inlining critical-path CSS would improve First Contentful Paint (FCP).
WP-15 Image Format Optimization Low index.html 27, 30, 33 WARN All images use .png format. Consider modern formats (WebP, AVIF) for 25-50% smaller file sizes and faster loading with <picture> fallback.
WP-16 Script Placement / Defer Medium index.html 37 WARN Inline script accesses DOM directly. If extracted to an external file, it should use the defer attribute to avoid blocking HTML parsing.
WP-17 XSS via innerHTML High index.html 42 FAIL innerHTML is set with unsanitized user input from location.hash. This is an XSS vulnerability and forces the browser to re-parse the DOM subtree, impacting performance.
WP-18 Third-Party Resource Impact Medium index.html 10 WARN External CDN resource (cdn.example.com/styles.css) introduces a single point of failure and additional DNS/TCP/TLS overhead. Consider self-hosting critical CSS assets.
