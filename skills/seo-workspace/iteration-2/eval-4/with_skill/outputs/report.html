수정 가이드

JavaScript 로딩 방식 이슈

WP-01: 렌더 블로킹 인라인 스크립트 (line 37-44)

문제: <script>가 body 내에서 동기적으로 실행되어 HTML 파싱을 차단합니다. 또한 innerHTML 사용으로 XSS 취약점이 존재합니다.

수정 방법:

<!-- 방법 1: defer 속성으로 외부 스크립트 분리 -->
<script defer src="/js/label-init.js"></script>

<!-- 방법 2: type="module"로 변경 (자동 defer) -->
<script type="module">
  const label = document.getElementById("label");
  if (label) {
    // textContent 사용으로 XSS 방지
    label.textContent = "Hello, " + decodeURIComponent(location.hash.slice(1));
  }
</script>

WP-03: 렌더 블로킹 외부 CSS (line 10)

문제: 외부 CSS 파일이 동기 로딩되어 렌더링을 차단합니다.

수정 방법:

<!-- Critical CSS 인라인 처리 -->
<style>/* 화면 상단(above-fold) 스타일 인라인 */</style>

<!-- 나머지 CSS 비동기 로딩 -->
<link rel="preload" href="https://cdn.example.com/styles.css"
      as="style" onload="this.onload=null;this.rel='stylesheet'">
<noscript><link rel="stylesheet" href="https://cdn.example.com/styles.css"></noscript>

폰트 최적화 이슈

WP-10: font-display 미설정 (line 13-17)

문제: @font-face에 font-display 속성이 없어 기본값 "auto"(대부분 "block"과 동일)로 동작합니다. 폰트 로딩 완료까지 텍스트가 보이지 않는 FOIT(Flash of Invisible Text) 현상이 발생하여 LCP 점수가 악화됩니다.

수정 방법:

@font-face {
  font-family: "CustomFont";
  src: url("/fonts/custom.woff2") format("woff2");
  font-display: swap;          /* 즉시 대체 폰트 표시, 로딩 후 교체 */
  size-adjust: 105%;           /* 대체 폰트 크기 매칭으로 CLS 최소화 */
  ascent-override: 95%;
  descent-override: 20%;
}

참고: CLS가 더 중요한 경우 font-display: optional 사용을 고려하세요. 캐시된 경우에만 커스텀 폰트를 사용하여 레이아웃 이동을 완전히 방지합니다.

WP-11: 중요 폰트 preload 미설정

문제: 커스텀 폰트 파일에 대한 preload 힌트가 없어 브라우저가 CSS 파싱 후에야 폰트 다운로드를 시작합니다. 폰트 발견이 지연되어 렌더링 속도가 저하됩니다.

수정 방법: head 태그 상단에 추가:

<link rel="preload" href="/fonts/custom.woff2"
      as="font" type="font/woff2" crossorigin>

Core Web Vitals 관련 이슈

WP-02: LCP 이미지 최적화 미흡 (line 27)

문제: hero 이미지가 LCP 요소일 가능성이 높으나 fetchpriority="high" 및 preload가 설정되어 있지 않습니다. 브라우저가 이미지의 우선순위를 자동으로 높이지 않아 LCP가 지연됩니다.

수정 방법:

<!-- head 태그에 preload 추가 -->
<link rel="preload" href="/public/hero.webp" as="image" fetchpriority="high">

<!-- img 태그에 fetchpriority 추가 -->
<img src="/public/hero.webp" alt="Hero banner"
     width="800" height="400" fetchpriority="high">

WP-04: CLS - 이미지 크기 미지정 (line 33)

문제: promo.png 이미지에 width/height 속성이 없어 이미지 로딩 시 레이아웃 이동(CLS)이 발생합니다.

수정 방법:

<img src="/public/promo.png" alt="Promotional banner"
     width="600" height="300">

<!-- 또는 CSS aspect-ratio 사용 -->
<img src="/public/promo.png" alt="Promotional banner"
     style="aspect-ratio: 2/1; width: 100%;">

WP-05: below-fold 이미지 lazy loading 미설정 (line 30, 33)

문제: LCP 이미지가 아닌 logo.png, promo.png에 loading="lazy"가 설정되어 있지 않아 불필요한 초기 리소스 로딩이 발생합니다.

수정 방법:

<img src="/public/logo.png" width="120" height="40"
     alt="Logo" loading="lazy" decoding="async">
<img src="/public/promo.png" alt="Promotional banner"
     width="600" height="300" loading="lazy" decoding="async">

WP-06: 레거시 이미지 포맷 사용

문제: 모든 이미지가 PNG 포맷을 사용합니다. WebP/AVIF 포맷으로 전환 시 30-50% 파일 크기 절감이 가능하여 LCP 개선에 직접적으로 기여합니다.

수정 방법:

<picture>
  <source type="image/avif" srcset="/public/hero.avif">
  <source type="image/webp" srcset="/public/hero.webp">
  <img src="/public/hero.png" alt="Hero banner"
       width="800" height="400" fetchpriority="high">
</picture>
