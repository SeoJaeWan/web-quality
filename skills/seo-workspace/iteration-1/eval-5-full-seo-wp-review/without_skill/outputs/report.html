Web Performance Issues (WP-01 ~ WP-18)

ID Severity File Line Issue Recommendation
WP-01 High index.html 10 Render-blocking external CSS loaded synchronously in <head>. This delays First Contentful Paint (FCP) until the stylesheet is downloaded and parsed. Use <link rel="preload" as="style"> with an onload handler, or inline critical CSS and load the rest asynchronously.
WP-02 High index.html 13-17 @font-face declaration is missing font-display: swap. This causes Flash of Invisible Text (FOIT) where text is hidden until the font loads. Add font-display: swap; to the @font-face rule so fallback text is visible during font loading.
WP-03 Medium index.html 33 Image promo.png is missing width and height attributes. This causes Cumulative Layout Shift (CLS) as the browser doesn't know the image dimensions until it loads. Always specify explicit width and height attributes on <img> tags to reserve layout space.
WP-04 Medium dashboard.tsx 50 Below-the-fold image is missing loading="lazy". Without lazy loading, offscreen images are fetched immediately, wasting bandwidth and delaying critical resources. Add loading="lazy" to images that are not in the initial viewport.
WP-05 Medium dashboard.tsx 10-14 Layout thrashing: offsetHeight is read (forcing reflow), then style.minHeight is written (triggering another reflow) in the same synchronous block. Batch DOM reads and writes separately. Use requestAnimationFrame for write operations after reading.
WP-06 Medium dashboard.tsx 17-19 Dynamic content inserted via setTimeout without any size reservation. When items appear after 500ms, the layout shifts, hurting CLS score. Reserve space with a skeleton placeholder or min-height on the container before dynamic content loads.
WP-07 Low index.html 37-44 Inline <script> in the body blocks HTML parsing. While at the end of body, it still runs synchronously and delays the DOMContentLoaded event. Move scripts to external files with the defer attribute, or ensure minimal inline script size.
WP-08 Medium index.html 27 Hero image uses PNG format without a modern format alternative. WebP and AVIF offer 25-50% smaller file sizes with similar quality. Use a <picture> element with WebP/AVIF sources and PNG fallback.
WP-09 Low index.html 27 Hero image lacks srcset and sizes attributes. All devices download the same large image regardless of screen size. Add srcset with multiple image sizes and a sizes attribute for responsive image delivery.
WP-10 Low index.html 10 External CSS loaded from cdn.example.com without preconnect resource hint. DNS lookup + TCP + TLS handshake adds latency. Add <link rel="preconnect" href="https://cdn.example.com"> early in the <head>.
WP-11 Medium Card.tsx 12 console.log left in production code. While the performance impact is small, it leaks information and adds unnecessary overhead in hot paths. Remove console.log from production code or use a configurable logger with environment-based levels.
WP-12 Low Card.tsx 17 document.querySelector used inside a React event handler instead of a ref. This bypasses React's virtual DOM, is slower, and can select unintended elements. Use useRef to reference the target element directly instead of querying the global DOM.
WP-13 Medium dashboard.tsx 45-47 List items rendered without a key prop. React cannot efficiently reconcile the list, causing unnecessary re-renders and potential state bugs. Add a stable, unique key prop to each item (avoid using array index if items can reorder).
WP-14 Low index.html — No evidence of HTTP caching strategy for static assets (CSS, images, fonts). Repeat visitors re-download assets unnecessarily. Configure Cache-Control headers with long max-age for hashed/versioned static assets.
WP-15 Medium index.html 42 innerHTML used with user-controlled input (location.hash). This is a direct XSS vulnerability and bypasses any framework protections. Use textContent instead of innerHTML, or sanitize user input with a library like DOMPurify.
WP-16 Low index.html 14 Custom font file loaded without a preload hint. The browser discovers the font only when parsing CSS, adding delay to text rendering. Add <link rel="preload" href="/fonts/custom.woff2" as="font" type="font/woff2" crossorigin> in <head>.
WP-17 Low dashboard.tsx — No evidence of code splitting or lazy loading for route-level components. The entire app bundle is loaded upfront. Use React.lazy() with Suspense for route-level code splitting to reduce initial bundle size.
WP-18 Low index.html 10 External CDN stylesheet loaded without Subresource Integrity (SRI). If the CDN is compromised, malicious CSS could be injected. Add integrity and crossorigin attributes to external resource tags for tamper detection.
