Image Optimization (5 issues)

HIGH dashboard.tsx:50

<img> element is missing width and height attributes. Without intrinsic dimensions the browser cannot reserve space during layout, directly causing Cumulative Layout Shift (CLS).

Fix: Add explicit width and height attributes that match the image's intrinsic aspect ratio. Example: <img src="/public/promo.png" alt="Promo" width="800" height="400" />. Alternatively, use the CSS aspect-ratio property on the element.
HIGH dashboard.tsx:50

Below-fold image lacks loading="lazy". The image is rendered after a long scrollable grid section (className="mt-8"), meaning it is almost certainly below the viewport on initial load.

Fix: Add loading="lazy" to the <img> tag. This defers the network request until the user scrolls near the image, saving bandwidth and improving Largest Contentful Paint (LCP) for above-fold content.
MEDIUM dashboard.tsx:50

Image is served as a single PNG with no modern format alternatives (WebP, AVIF). Modern formats reduce file size by 25-50% compared to PNG.

Fix: Use a <picture> element with <source> tags for AVIF and WebP, falling back to PNG. Or adopt the Next.js <Image> component which handles format negotiation automatically.
MEDIUM dashboard.tsx:50

No srcset or sizes attributes for responsive image serving. Mobile devices may download an image far larger than needed.

Fix: Provide multiple resolutions via srcset and define sizes to match your layout breakpoints.
MEDIUM dashboard.tsx:50

No fetchpriority attribute. For a below-fold promotional image, the browser may still fetch it with default priority, competing with critical above-fold resources.

Fix: Add fetchpriority="low" to deprioritize the below-fold image and let the browser focus bandwidth on above-fold content.

CLS Prevention (3 issues)

HIGH dashboard.tsx:16-19

Dynamic content is injected after a 500ms setTimeout with no space reserved in the layout. When the three list items appear, every element below them shifts downward -- a textbook CLS violation.

Fix: Reserve space before the async data arrives. Options: (1) set a CSS min-height on the grid container that accounts for the expected items, (2) render skeleton/placeholder elements immediately that are later replaced with real data, or (3) use CSS contain: layout on the container to prevent its children from affecting external layout.
MEDIUM dashboard.tsx:41

The grid container (<section>) has no explicit dimensions or skeleton state. Its height is entirely determined by its children, which arrive asynchronously.

Fix: Add a min-height style or render placeholder skeleton cards so the layout is stable before real content loads.
MEDIUM Card.tsx:27

All Card instances share the same id="card-item". Duplicate IDs can cause unpredictable behavior in assistive technology and in any JavaScript that queries by ID, potentially leading to layout or interaction bugs.

Fix: Generate unique IDs per Card instance (e.g., id={`card-${title.toLowerCase().replace(/\s+/g, '-')}`}).

Rendering Performance (4 issues)

HIGH dashboard.tsx:10-14

Layout thrashing in useEffect: reading offsetHeight (forces a synchronous reflow) then immediately writing style.minHeight (invalidates layout, triggers another reflow). This forced-reflow cycle degrades frame rate and delays First Contentful Paint.

Fix: Separate reads and writes. Use requestAnimationFrame to schedule the write after the read:
const h = containerRef.current.offsetHeight;
requestAnimationFrame(() => { containerRef.current.style.minHeight = h + "px"; });
Or better yet, achieve the same result with pure CSS (min-height: fit-content).
MEDIUM dashboard.tsx:45-47

List items rendered via .map() without a key prop. React cannot efficiently reconcile the list and may re-render all items instead of just the changed ones.

Fix: Add a stable, unique key: {items.map((item) => <div key={item} ...>)}. If items can have duplicates, use a unique identifier or combine with the index.
LOW Card.tsx:17-20

Direct DOM manipulation via document.querySelector inside a React component bypasses the virtual DOM, creating a mismatch between React's internal state and the actual DOM.

Fix: Use a useRef to reference the target element and update its style through state or the ref, staying within React's rendering model.
LOW Card.tsx:13

console.log left in an event handler. While the performance impact is minor, it adds unnecessary overhead in production and can slow down the event loop if triggered frequently.

Fix: Remove the console.log or guard it with a development-only check (e.g., if (process.env.NODE_ENV === 'development')).
