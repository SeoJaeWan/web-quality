Correction Guide

WP-04 (Fail): Image missing width/height — CLS prevention

File: dashboard.tsx line 50

Current:

<img src="/public/promo.png" alt="Promo" className="mt-8" />

Fix:

<img src="/public/promo.png" alt="Promo" className="mt-8"
     width={800} height={400} />

Or use CSS aspect-ratio:

<img src="/public/promo.png" alt="Promo"
     className="mt-8 w-full" style={{ aspectRatio: '2/1' }} />

WP-05 (Fail): Below-fold image without lazy loading

File: dashboard.tsx line 50

Fix: Add loading="lazy" and decoding="async":

<img src="/public/promo.png" alt="Promo" className="mt-8"
     width={800} height={400} loading="lazy" decoding="async" />

WP-06 (Fail): Legacy image format (.png)

File: dashboard.tsx line 50

Fix: Convert to WebP/AVIF and use <picture> for format fallback:

  <source type="image/avif" srcSet="/public/promo.avif" />

  <img src="/public/promo.png" alt="Promo" className="mt-8"
       width={800} height={400} loading="lazy" decoding="async" />
</picture>

WP-09 (Fail): Layout thrashing in useEffect

File: dashboard.tsx lines 12-13

Current:

const h = containerRef.current.offsetHeight;
containerRef.current.style.minHeight = h + "px";

Fix: Use requestAnimationFrame to batch reads and writes, or use CSS min-height directly:

// Option 1: Use CSS instead
// Remove the JS measurement entirely; set min-height via CSS class

// Option 2: If measurement is needed, batch with rAF
const h = containerRef.current.offsetHeight;
requestAnimationFrame(() => {
  containerRef.current.style.minHeight = h + "px";
});
