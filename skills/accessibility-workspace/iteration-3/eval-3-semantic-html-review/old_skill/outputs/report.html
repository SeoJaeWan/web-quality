항목 11. 초점 이동과 표시 — SearchFilter.tsx:26

문제: 검색 입력란에 focus:outline-none Tailwind 클래스가 적용되어 키보드 포커스 시 아웃라인이 제거됨. 필터 버튼들에도 별도 포커스 스타일 없음.

수정: focus:outline-none을 focus:ring-2 focus:ring-blue-500 focus:ring-offset-1으로 교체. 필터 버튼에도 동일한 포커스 스타일 추가.

항목 17. 반복 영역 건너뛰기 — Navbar.tsx (전체)

문제: 페이지 최상단에 본문 바로가기 링크가 없어 키보드/스크린리더 사용자가 반복 내비게이션을 매번 탐색해야 함.

수정: <a href="#main-content" className="sr-only focus:not-sr-only focus:absolute focus:top-2 focus:left-2 focus:z-50 focus:bg-white focus:px-4 focus:py-2">본문 바로가기</a> 를 <nav> 이전에 추가. 메인 콘텐츠 영역에 id="main-content" 추가.

항목 29. 레이블 제공 — SearchFilter.tsx:21-28

문제: 검색 입력란(<input type="search">)에 <label>, aria-label, title 속성 없음. placeholder만 있으며 이는 KWCAG2.2에서 레이블로 인정하지 않음.

수정: <input type="search" aria-label="할 일 검색" ...>

항목 33. 웹 애플리케이션 접근성 준수 — Navbar.tsx:59-72, SearchFilter.tsx:32-44

문제 1 (Navbar): 햄버거 버튼 클릭 시 모바일 메뉴가 열리지만 aria-expanded 상태 속성이 변경되지 않음. 스크린리더 사용자가 메뉴 열림/닫힘 상태 인식 불가.

수정 1: <button aria-label={menuOpen ? "메뉴 닫기" : "메뉴 열기"} aria-expanded={menuOpen} aria-controls="mobile-menu" ...>
모바일 메뉴 div에 id="mobile-menu" 추가.

문제 2 (SearchFilter): 필터 버튼(전체/진행중/완료) 클릭 시 CSS 클래스는 변경되지만 aria-pressed 상태 속성 없음. 스크린리더 사용자가 현재 선택된 필터 인식 불가.

수정 2: <button aria-pressed={filter === f.value} ...>
