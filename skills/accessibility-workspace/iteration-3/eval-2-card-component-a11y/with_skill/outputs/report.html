항목 10. 키보드 사용 보장 (A-10) — StatsCard.tsx:10,17

문제: <div onClick={handleClick}>가 키보드로 접근 불가. tabIndex, role, onKeyDown 모두 없음. 마우스 없는 사용자는 기능 사용 불가.

수정 방법 1 (권장) — button으로 교체:

<button
  id={`stat-card-${testId}`}
  className="rounded-xl border border-zinc-200 bg-white p-6 dark:border-zinc-800 dark:bg-zinc-900 text-left w-full"
  data-testid={testId}
  onClick={handleClick}
  aria-label={`${label}: ${value}`}
>
  <p className="text-3xl font-bold text-zinc-900 dark:text-zinc-50">{value}</p>
  <p className="mt-1 text-sm text-zinc-500">{label}</p>
</button>

수정 방법 2 — ARIA 속성 추가 (button 교체 불가 시):

<div
  role="button"
  tabIndex={0}
  aria-label={`${label}: ${value}`}
  onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}
  onClick={handleClick}
  ...
>

항목 23. 레이블과 네임 (A-23) — StatsCard.tsx:10

문제: 클릭 가능한 <div>에 접근 가능한 네임(name)이 없음. 스크린 리더가 이 요소를 식별할 수 없음. 시각적으로 보이는 {label} 텍스트가 보조기술에 연결되지 않음.

수정: aria-label={`${label}: ${value}`} 추가 (또는 위 A-10 수정 방법의 button 교체로 자동 해결)

문제: id="stat-card" 하드코딩. 재사용 컴포넌트에 고정 id를 사용하면 복수 렌더링 시 동일 id 중복 발생 — HTML 유일성 요건 위반, KWCAG2.2 A-32 위반.

// id 제거 (권장)
<div
  className="..."

// 또는 고유 id 사용
id={`stat-card-${testId}`}

항목 33. 웹 애플리케이션 접근성 준수 (A-33) — StatsCard.tsx:10,17

문제: 클릭 가능한 커스텀 컴포넌트(<div onClick>)에 WAI-ARIA role이 없음. 스크린 리더는 이를 단순 컨테이너로 인식하여 상호작용 가능 여부를 알 수 없음. 보조기술 사용자에게 기능 접근 불가.

수정: A-10 수정 방법과 동일 — <button>으로 교체하거나 role="button" + aria-label 추가
