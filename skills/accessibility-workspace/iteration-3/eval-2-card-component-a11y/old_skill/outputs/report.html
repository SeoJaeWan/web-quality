불합격 (❌)

항목 10. 키보드 사용 보장 (A-10) — StatsCard.tsx:10-18

문제: <div onClick={handleClick}>에 키보드 이벤트 없음. tabIndex가 -1이므로 Tab 키로 접근 불가. Playwright로 런타임 확인 완료.

수정:

방법 1 (권장) — <button>으로 교체:

<button type="button" onClick={handleClick} className="rounded-xl border border-zinc-200 bg-white p-6 ...">

방법 2 — ARIA 속성 추가:

<div role="button" tabIndex={0} onClick={handleClick} onKeyDown={(e) => (e.key === 'Enter' || e.key === ' ') && handleClick()} ...>

항목 23. 레이블과 네임 (A-23) — StatsCard.tsx:10

문제: 클릭 가능한 div에 접근 가능한 이름(accessible name) 없음. aria-label, aria-labelledby, role 모두 누락. 스크린리더 사용자에게 카드 기능 안내 불가.

수정: aria-label={`${label}: ${value}건 — 클릭하여 상세 보기`} 추가 (또는 button 교체 시 텍스트 콘텐츠로 대체됨)

항목 32. 마크업 오류 방지 (A-32) — StatsCard.tsx:13

문제: id="stat-card" 하드코딩. 컴포넌트가 3회 렌더링될 때 동일 id 3개 생성. Playwright 런타임 확인: duplicates: ["stat-card"]. HTML 명세 위반 (id는 페이지 내 유일해야 함).

수정: id={`stat-card-${testId}`} — testId prop이 이미 고유 값(stat-total, stat-completed, stat-pending)으로 전달되므로 활용

항목 33. 웹 애플리케이션 접근성 (A-33) — StatsCard.tsx:10-18

문제: 커스텀 대화형 컴포넌트(div with onClick)에 WAI-ARIA role, 상태 속성 미제공. 스크린리더가 interactive 요소로 인식 불가.

수정:

<div role="button" aria-label={`${label}: ${value}건`} tabIndex={0} onClick={handleClick} onKeyDown={...}>

또는 <button>으로 교체 (role 자동 부여됨)

시맨틱 HTML: 대화형 요소 — StatsCard.tsx:10-18

문제: onClick을 가진 <div>. 키보드 접근성, ARIA, 시맨틱 의미 모두 결여.

수정: <button type="button">으로 교체 — 브라우저가 자동으로 role="button", 키보드 포커스, Enter/Space 처리 제공
