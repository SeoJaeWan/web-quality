## 원칙 2. 운용의 용이성 (Operable)

### 10. 키보드 사용 보장 — ❌ 위반

- **판정방식:** 정적분석
- **발견된 문제:**
  - `Card.tsx:24-30` — `<div onClick={handleClick}>` 로 클릭 이벤트를 처리하지만, `<div>`는 비대화형 요소로서 키보드 접근 불가. `tabIndex`, `role="button"`, `onKeyDown` 핸들러가 모두 누락됨.
- **수정 가이드:**
  - `<div>` 대신 `<button>` 사용 권장
  - 또는 `role="button"` + `tabIndex={0}` + `onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}` 추가

---

### 11. 초점 이동과 표시 — ⚠️ 권고

- **판정방식:** 정적분석
- **발견된 문제:**
  - 명시적인 `outline: none` 또는 `outline: 0` 패턴은 발견되지 않음.
  - 그러나 `Card.tsx`의 클릭 가능한 `<div>`에 포커스 스타일이 전혀 정의되어 있지 않으므로, 키보드 사용 시 초점 표시가 되지 않을 가능성이 높음.
- **수정 가이드:** 클릭 가능한 요소에 `focus-visible` 스타일 또는 `outline` 스타일 추가

## 원칙 4. 견고성 (Robust)

### 32. 마크업 오류 방지 — ❌ 위반

- **판정방식:** 정적분석
- **발견된 문제:**
  - `Card.tsx:27` — `id="card-item"`이 하드코딩되어 있어, `Card` 컴포넌트가 리스트에서 복수 렌더링될 경우 동일 `id`가 중복됨. HTML 명세상 `id`는 페이지 내 유일해야 함.
  - `dashboard.tsx:45-46` — `items.map((item) => (<div ...>{item}</div>))` 에서 `key` prop이 누락됨. React 리스트 렌더링 시 `key`는 필수.
- **수정 가이드:**
  - `Card.tsx`: `id="card-item"` 제거하거나 고유한 동적 ID 생성 (예: `id={`card-${title}`}`)
  - `dashboard.tsx`: `items.map((item, index) => (<div key={item} ...>{item}</div>))`

---

### 33. 웹 애플리케이션 접근성 준수 — ❌ 위반

- **판정방식:** 정적분석
- **발견된 문제:**
  - `Card.tsx:24-30` — `<div onClick>` 로 구현된 커스텀 인터랙티브 컴포넌트에 `role`, `aria-*` 상태 속성이 전혀 없음. 스크린 리더에 이 요소가 상호작용 가능하다는 정보가 전달되지 않음.
  - `dashboard.tsx:29-36` — 아이콘 버튼에 `aria-label`이 없어, WAI-ARIA 접근 가능한 이름(accessible name)이 부재.
- **수정 가이드:**
  - `Card.tsx`: `<div>` → `<button>` 변경 또는 `role="button"` + `tabIndex={0}` + 적절한 `aria-label` 추가
  - `dashboard.tsx`: 아이콘 버튼에 `aria-label="기능 설명"` 추가
