2. 시맨틱 HTML 구조 검토

#이슈심각도라인설명KWCAG 2.2 관련 항목개선 방안
S-01 시맨틱 랜드마크 요소 미사용 주요 20-45 페이지에 <header>, <nav>, <main>, <footer> 등의 시맨틱 랜드마크 요소가 전혀 사용되지 않았습니다. 모든 콘텐츠가 <body> 직접 하위에 나열되어 있어 스크린 리더 사용자가 페이지 구조를 파악하기 어렵습니다. 4.1.2 (마크업 오류 방지), 2.4.1 (반복 영역 건너뛰기) 콘텐츠를 <header>, <nav>, <main>, <footer> 등으로 구조화하세요. 주요 콘텐츠를 <main> 태그로 감싸세요.
S-02 네비게이션 영역 미구분 주요 24 "About Us" 링크가 <nav> 요소 없이 단독으로 존재합니다. 네비게이션 링크는 <nav> 랜드마크로 감싸야 보조 기술에서 탐색 영역으로 인식합니다. 2.4.1 (반복 영역 건너뛰기), 1.3.1 (정보와 관계) <nav aria-label="메인 네비게이션"> 요소로 링크를 감싸세요.
S-03 콘텐츠 섹션 미구분 경미 21-35 제목(<h1>), 이미지, 텍스트 등 다양한 콘텐츠가 섹션 구분 없이 나열되어 있습니다. <section> 또는 <article> 태그를 사용하여 논리적 단위로 구분하면 접근성이 향상됩니다. 1.3.1 (정보와 관계) 관련 콘텐츠를 <section>으로 그룹화하고 적절한 헤딩을 부여하세요.
S-04 문서 언어와 실제 콘텐츠 불일치 가능성 정보 2 <html lang="en">으로 영어 문서로 선언되어 있습니다. 한국어 콘텐츠가 포함될 경우 lang="ko"로 변경하거나 해당 요소에 lang 속성을 지정해야 합니다. 3.1.1 (기본 언어 표시) 실제 콘텐츠 언어에 맞게 lang 속성을 설정하세요.

3. 접근성 (Accessibility) 검토

#이슈심각도라인설명KWCAG 2.2 관련 항목개선 방안
A-01 이미지 alt 속성 누락 심각 30 <img src="/public/logo.png">에 alt 속성이 없습니다. 스크린 리더 사용자가 이미지의 내용을 전혀 알 수 없으며, KWCAG 2.2의 필수 요구사항을 위반합니다. 1.1.1 (적절한 대체 텍스트 제공) alt="회사 로고" 등 이미지 목적을 설명하는 대체 텍스트를 추가하세요. 장식용 이미지라면 alt=""와 role="presentation"을 사용하세요.
A-02 건너뛰기 링크(Skip Navigation) 미제공 심각 20 페이지 상단에 키보드 사용자가 반복 영역을 건너뛰고 주요 콘텐츠로 이동할 수 있는 건너뛰기 링크가 없습니다. 2.4.1 (반복 영역 건너뛰기) <body> 바로 뒤에 <a href="#main-content" class="skip-link">본문 바로가기</a>를 추가하고, 주요 콘텐츠 영역에 id="main-content"를 부여하세요.
A-03 링크 텍스트 맥락 부족 경미 35 <a href="...">latest deals</a> 링크의 텍스트가 주변 문맥 없이는 목적을 명확히 알기 어려울 수 있습니다. 독립적으로 읽었을 때 링크의 목적을 이해할 수 있어야 합니다. 2.4.4 (링크 텍스트) 링크 텍스트를 더 구체적으로 변경하거나, aria-label을 활용하여 링크 목적을 명확히 하세요.
A-04 innerHTML을 통한 XSS 취약점 및 접근성 문제 심각 37-44 innerHTML으로 사용자 입력(location.hash)을 직접 삽입하고 있어 XSS 공격에 취약합니다. 동적으로 삽입된 콘텐츠가 스크린 리더에 적절히 전달되지 않을 수 있습니다. 4.1.2 (마크업 오류 방지) textContent를 사용하고, 동적 콘텐츠 변경 시 aria-live="polite" 영역을 활용하여 보조 기술에 변경 사항을 알리세요.
A-05 meta description 누락 주요 7 <meta name="description">이 없어 검색 엔진과 보조 기술이 페이지 내용을 요약할 수 없습니다. - <meta name="description" content="페이지 설명">을 <head>에 추가하세요.
A-06 이미지 width/height 누락 (CLS 위험) 경미 33 <img src="/public/promo.png">에 width와 height 속성이 없어 레이아웃 시프트(CLS)가 발생할 수 있습니다. 시각적 불안정은 인지 장애 사용자에게 혼란을 줄 수 있습니다. - 이미지에 width와 height 속성을 명시하거나 CSS aspect-ratio를 사용하세요.
A-07 font-display 미설정 주요 13-17 @font-face에 font-display: swap이 설정되지 않아 폰트 로딩 중 텍스트가 보이지 않을 수 있습니다(FOIT). 저시력 사용자나 저속 네트워크 환경에서 콘텐츠 접근이 지연됩니다. - font-display: swap;을 @font-face 규칙에 추가하세요.
A-08 페이지 제목 구체성 부족 정보 6 "Sample Page"라는 제목은 페이지의 목적을 구체적으로 설명하지 못합니다. 접근성 측면에서 페이지 제목은 내용을 명확히 나타내야 합니다. 2.4.2 (제목 제공) 페이지의 실제 목적을 반영하는 구체적인 제목을 사용하세요.

<!DOCTYPE html> <html lang="ko"> <head> <meta charset="UTF-8" /> <meta name="viewport" content="width=device-width, initial-scale=1.0" /> <title>회사명 - 메인 페이지</title> <meta name="description" content="회사의 주요 서비스와 최신 프로모션을 확인하세요." /> <link rel="stylesheet" href="https://cdn.example.com/styles.css" media="print" onload="this.media='all'" /> <style> @font-face { font-family: "CustomFont"; src: url("/fonts/custom.woff2") format("woff2"); font-display: swap; } .skip-link { position: absolute; left: -9999px; } .skip-link:focus { position: static; } </style> </head> <body> <a href="#main-content" class="skip-link">본문 바로가기</a> <header> <h1>Welcome to the Sample Page</h1> <nav aria-label="메인 네비게이션"> <a href="https://example.com/about">About Us</a> </nav> </header> <main id="main-content"> <section aria-labelledby="hero-heading"> <h2 id="hero-heading" class="visually-hidden">메인 배너</h2> <img src="/public/hero.png" alt="Hero banner" width="800" height="400" /> <img src="/public/logo.png" alt="회사 로고" width="120" height="40" /> <img src="/public/promo.png" alt="Promotional banner" width="600" height="300" /> </section> <section> <p>Check out our <a href="https://example.com/deals">최신 할인 프로모션 보기</a>.</p> <div id="label" aria-live="polite"></div> </section> </main> <footer> <p>&copy; 2026 Company Name</p> </footer> <script> var label = document.getElementById("label"); var userInput = location.hash.slice(1); if (label) { label.textContent = "Hello, " + decodeURIComponent(userInput); } </script> </body> </html>

5. 검토 결론

index.html 페이지는 기본적인 HTML 문서 구조(<!DOCTYPE html>, lang 속성, charset)는 갖추고 있으나, 시맨틱 랜드마크 요소가 전혀 사용되지 않아 보조 기술 사용자의 페이지 탐색에 심각한 제한이 있습니다.

가장 시급한 개선 사항:

  • alt 속성 누락 (라인 30) - KWCAG 2.2 1.1.1 위반
  • 건너뛰기 링크 미제공 - KWCAG 2.2 2.4.1 위반
  • innerHTML XSS 취약점 (라인 42) - 보안 및 접근성 동시 위험
  • 시맨틱 랜드마크 요소 도입 - header, nav, main, footer 구조화 필요
