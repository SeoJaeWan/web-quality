- **결과:** ❌ 위반
- **판정 방식:** 정적분석
- **발견된 문제:** `Card.tsx:24-29` — `<div>` 요소에 `onClick={handleClick}` 핸들러가 있으나, 키보드 접근을 위한 `tabIndex`, `role`, `onKeyDown` 등이 전혀 없음. `<div>`는 비대화형 요소이므로 키보드로 포커스 및 활성화가 불가능함.
- **수정 가이드:**
  - 방법 1 (권장): `<div>`를 `<button>`으로 변경하여 네이티브 키보드 접근성 확보
  - 방법 2: `<div>` 에 `role="button"`, `tabIndex={0}`, `onKeyDown` (Enter/Space 처리) 추가
  ```tsx
  <div
    role="button"
    tabIndex={0}
    onClick={handleClick}
    onKeyDown={(e) => {
      if (e.key === 'Enter' || e.key === ' ') {
        e.preventDefault();
        handleClick();
      }
    }}
  >
  ```

## 원칙 4. 견고성 (Robust)

### Item 32. 마크업 오류 방지
- **결과:** ❌ 위반
- **판정 방식:** 정적분석
- **발견된 문제:** `Card.tsx:27` — `id="card-item"`이 하드코딩되어 있음. 이 컴포넌트가 리스트 내에서 여러 번 렌더링될 경우, 동일한 `id` 값이 페이지 내에 중복 생성되어 KWCAG2.2 항목 32-4 (동일 ID 속성값 중복 사용) 위반.
- **수정 가이드:**
  - `id`를 고유하게 생성하거나, 필요 없다면 제거:
  ```tsx
  // 방법 1: id 제거
  <div className={`rounded border p-4 ${className}`} onClick={handleClick}>

  // 방법 2: 고유 id 사용
  <div id={`card-${title.replace(/\s+/g, '-').toLowerCase()}`} ...>

  // 방법 3: React useId 훅 사용
  const id = useId();
  <div id={`card-${id}`} ...>
  ```

### Item 33. 웹 애플리케이션 접근성 준수
- **결과:** ❌ 위반

  - `aria-label`로 접근 가능한 이름 제공

## 종합 수정 가이드

가장 높은 우선순위로 수정해야 할 사항:

### 1. `<div onClick>` 키보드 접근성 확보 (Item 10, 33 동시 해결)

```tsx
// Before (위반)
<div
  id="card-item"
  className={`rounded border p-4 ${className}`}
  onClick={handleClick}
>

// After (수정)
<div
  role="button"
  tabIndex={0}
  className={`rounded border p-4 cursor-pointer focus:ring-2 focus:ring-blue-500 focus:outline-none ${className}`}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleClick();
    }
  }}
  aria-label={`${title} 카드`}
>
```

### 2. 중복 ID 제거 (Item 32 해결)

### 3. 시맨틱 요소 교체 (권고)

카드 전체가 클릭 가능한 패턴이라면, `<button>` 요소로 교체하면 Item 10, 11, 23, 33, 시맨틱 HTML 문제가 모두 해결됨:

```tsx
<button
  type="button"

  <div className="card-highlight">{children}</div>
</button>
```
