#항목명결과판정방식발견된 문제수정 가이드

랜드마크 요소 (header/main/footer/nav) ✅ 정적분석 개별 카드 컴포넌트 — 랜드마크는 페이지 수준에서 판단
섹션 구조 (article/section/aside) ⚠️ 정적분석 Card.tsx:24 — 카드는 독립적인 콘텐츠 블록으로 <article> 요소가 더 적합할 수 있음 최상위 <div>를 <article>로 변경 권고. 카드가 독립적으로 의미 있는 콘텐츠 단위라면 <article>이 시맨틱적으로 적합
대화형 요소 (button/a vs div/span) ❌ 정적분석 Card.tsx:24-30 — 클릭 가능한 <div onClick>이 <button> 또는 <a>로 대체 가능 카드 전체가 클릭 가능하다면 <button>으로 감싸거나, 내부에 <a> 링크를 추가하여 접근성 확보. 또는 카드 내부에 별도의 <button> 액션 요소 배치
목록 구조 (ul/ol/li) ✅ 정적분석 반복 항목 구조 없음 (개별 컴포넌트)
표현용 태그 (b/i vs strong/em) ➖ 정적분석 <b>, <i> 태그 미사용
폼 그룹 (fieldset/legend) ➖ 정적분석 폼 요소 없음
기타 시맨틱 요소 (time/blockquote 등) ➖ 정적분석 날짜/인용문 콘텐츠 없음

불합격 (❌)

항목 11. 초점 이동과 표시 — Card.tsx:24-30

문제: 클릭 가능한 <div onClick={handleClick}>에 tabIndex가 없어 키보드 초점을 받을 수 없음. 초점 표시 스타일도 미정의

수정: <div tabIndex={0} className="... focus:outline-2 focus:outline-blue-500" ...> 또는 <button> 요소로 변경

항목 32. 마크업 오류 방지 — Card.tsx:27

문제: id="card-item"이 하드코딩되어 있어, 리스트 렌더링 시 동일 id가 중복됨 (HTML 유효성 위반)

수정: id={`card-${useId()}`} 또는 id 속성 제거

항목 33. 웹 애플리케이션 접근성 — Card.tsx:24-30

문제: 클릭 가능한 커스텀 <div>에 WAI-ARIA role 속성이 없음. 스크린 리더가 이 요소를 대화형 컨트롤로 인식할 수 없음

수정: <div role="button" tabIndex={0} aria-label={title} onKeyDown={handleKeyDown} onClick={handleClick}> 또는 <button> 요소로 변경

시맨틱 HTML: 대화형 요소 — Card.tsx:24-30

문제: 클릭 가능한 <div onClick>이 <button> 또는 <a>로 대체 가능

수정: 카드 전체 클릭이 필요하면 <button>으로 변경하거나, 카드 내부에 별도 <a>/<button> 배치
