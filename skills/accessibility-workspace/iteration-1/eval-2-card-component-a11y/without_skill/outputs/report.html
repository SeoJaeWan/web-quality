Card.tsx KWCAG 2.2 접근성 검토 보고서

파일: features/web-sample/components/Card.tsx | 검토일: 2026-03-11 | 기준: KWCAG 2.2 (한국형 웹 콘텐츠 접근성 지침 2.2)

검토 요약

Card 컴포넌트에서 총 7건의 접근성 문제가 발견되었습니다.

심각 (Critical): 3건 주요 (Major): 3건 보통 (Minor): 1건
# KWCAG 항목 심각도 라인 문제 설명 개선 방안
1 5.1.1 키보드 접근성
(운용의 용이성)		 심각 24-30 <div> 요소에 onClick 핸들러만 부여되어 있고 onKeyDown/onKeyUp 핸들러가 없습니다. 키보드 사용자는 이 카드를 클릭할 수 없습니다. <div>는 기본적으로 포커스를 받지 않으므로 tabIndex도 지정되어 있지 않아 탭 키로도 접근이 불가합니다.
방법 1 (권장): 클릭 가능한 요소라면 <button> 또는 <a> 등 인터랙티브 HTML 요소로 변경합니다.
방법 2: role="button", tabIndex={0}, onKeyDown (Enter/Space 처리)을 함께 추가합니다. 
<div
  role="button"
  tabIndex={0}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      handleClick();
    }
  }}
>
2 5.1.1 키보드 접근성
(초점 이동)		 심각 24 <div>에 tabIndex가 없어 키보드 탭 순서에서 완전히 제외됩니다. 클릭 가능한 요소임에도 키보드로 포커스를 줄 수 없으므로 키보드 전용 사용자가 해당 카드와 상호작용할 수 없습니다. tabIndex={0}을 추가하여 자연스러운 탭 순서에 포함시킵니다. 동시에 :focus-visible 등의 스타일로 포커스 표시기를 제공해야 합니다.
3 5.3.1 적절한 역할 부여
(인식의 용이성)		 심각 24 클릭 가능한 <div>에 ARIA role 속성이 없습니다. 스크린 리더 사용자는 이 요소가 클릭 가능한 버튼인지, 단순한 텍스트 컨테이너인지 구분할 수 없습니다. 보조 기술이 요소의 목적과 상호작용 방법을 전달하지 못합니다. role="button" 또는 시맨틱 <button> 요소를 사용합니다. 카드가 링크 역할이라면 <a> 또는 role="link"를 사용합니다.
4 5.1.3 초점의 시각화
(운용의 용이성)		 주요 24-30 포커스를 받을 수 있게 수정하더라도 현재 코드에는 포커스 표시(focus indicator)를 위한 스타일이 전혀 정의되어 있지 않습니다. Tailwind CSS의 rounded border p-4만 적용된 상태이므로, 브라우저 기본 포커스 아웃라인이 outline: none 등으로 제거될 가능성이 있습니다. Tailwind의 focus:ring-2 focus:ring-blue-500 focus-visible:outline-2 등을 추가하여 포커스 상태가 시각적으로 명확히 표시되도록 합니다. 
className={`rounded border p-4 focus:ring-2 focus:ring-blue-500 focus:outline-none ${className}`}
5 5.3.3 콘텐츠의 선형화
(이해의 용이성)		 주요 25 id="card-item"이 하드코딩되어 있어, 카드가 리스트에서 여러 번 렌더링될 경우 동일한 id가 중복됩니다. 중복 ID는 HTML 명세 위반이며, 보조 기술(스크린 리더)이 특정 요소를 정확히 참조하지 못하게 합니다. aria-labelledby, aria-describedby 등이 ID를 참조할 때 잘못된 요소를 가리킬 수 있습니다. 고유한 ID를 생성하도록 변경합니다. React 18+에서는 useId() 훅을 사용할 수 있습니다. 
const id = React.useId();
// ...
<div id={`card-${id}`} ...>
6 5.2.1 접근 가능한 이름
(인식의 용이성)		 주요 24 클릭 가능한 <div>에 접근 가능한 이름(accessible name)이 제공되지 않았습니다. aria-label 또는 aria-labelledby가 없으므로 스크린 리더 사용자는 이 요소의 목적을 파악하기 어렵습니다. 내부의 <h3>에 제목이 있지만 프로그래밍적으로 연결되어 있지 않습니다. aria-label 또는 aria-labelledby를 사용하여 접근 가능한 이름을 제공합니다. 
<h3 id={`card-title-${id}`} ...>{title}</h3>
// div에 추가:
aria-labelledby={`card-title-${id}`}
7 5.1.2 조작 가능
(운용의 용이성 - 충분한 크기)		 보통 28 클릭 가능한 카드 요소에 대해 최소 터치/클릭 영역 크기가 CSS에서 명시적으로 보장되지 않습니다. p-4(padding 1rem)이 적용되어 있어 어느 정도의 크기는 확보되지만, 콘텐츠가 매우 적을 경우 WCAG 2.2의 Target Size 기준(최소 24x24 CSS px)을 충족하지 못할 수 있습니다. min-height와 min-width를 명시적으로 설정하여 최소 클릭 영역을 보장합니다. 
className={`rounded border p-4 min-h-[44px] ${className}`}
