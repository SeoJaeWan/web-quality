불합격 (❌)

항목 10. 키보드 사용 보장 — StatsCard.tsx:10-17

문제: <div onClick={handleClick}>로 구현된 클릭 가능 요소에 tabIndex, role, onKeyDown이 없어 키보드로 접근 및 조작이 불가능합니다. Playwright Tab 키 탐색에서 StatsCard 요소에 포커스가 전혀 도달하지 않았습니다.

수정: <div> 대신 <button> 사용을 권장합니다. 또는 role="button" tabIndex={0} onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}를 추가하세요.

항목 32. 마크업 오류 방지 — StatsCard.tsx:12

문제: id="stat-card"가 하드코딩되어 리스트 렌더링 시 동일 id가 3회 중복됩니다. HTML 사양에서 id는 문서 내 고유해야 합니다. Playwright에서 document.querySelectorAll('#stat-card').length === 3으로 확인되었습니다.

수정: 고정 id를 제거하거나 id={`stat-card-${testId}`}처럼 고유 id를 생성하세요.

권고 (⚠️)

항목 23. 레이블과 네임

내용: 클릭 가능한 <div>에 aria-label이 없어 보조 기술 사용자가 요소의 용도를 파악할 수 없습니다. aria-label={label}을 추가하여 접근 가능한 이름을 제공하세요.

항목 33. 웹 애플리케이션 접근성

내용: 커스텀 클릭 가능 <div>에 WAI-ARIA role/state가 전혀 없습니다. role="button"을 추가하거나 네이티브 <button>을 사용하세요.

시맨틱 HTML: 섹션 구조

내용: 독립적인 통계 카드 콘텐츠를 <div>로 구현하고 있습니다. <article> 또는 <section aria-label={label}>으로 변경하면 시맨틱 의미가 향상됩니다.
