항목 10. 키보드 사용 보장 — StatsCard.tsx:10-17

문제: <div onClick={handleClick}>에 키보드 접근 수단(tabIndex, onKeyDown)이 없어 키보드 사용자가 이 요소를 활성화할 수 없음.

수정: <div>를 <button>으로 변경 (권장):

<button
  type="button"
  className="rounded-xl border border-zinc-200 bg-white p-6 dark:border-zinc-800 dark:bg-zinc-900 text-left w-full"
  data-testid={testId}
  onClick={handleClick}
>
  <p className="text-3xl font-bold text-zinc-900 dark:text-zinc-50">{value}</p>
  <p className="mt-1 text-sm text-zinc-500">{label}</p>
</button>

문제: id="stat-card"가 하드코딩되어 있어, 컴포넌트가 여러 번 렌더링되면 동일 ID가 중복됨. dashboard/page.tsx에서 3회 사용되므로 실제로 중복 발생.

// 방법 1: testId를 활용한 고유 ID
id={`stat-card-${testId}`}

// 방법 2: id 속성 제거 (data-testid로 충분한 경우)
// id 속성 삭제

시맨틱 HTML: 대화형 요소 — StatsCard.tsx:10

문제: 클릭 가능한 <div>를 사용 중. <button>으로 대체하면 키보드 접근성(A-10), ARIA(A-33), 레이블(A-23) 문제가 동시에 해결됨.

수정: 위 항목 10 수정안 참조
