시맨틱 HTML 검토

검토 항목결과판정방식발견된 문제권고 수정
랜드마크 요소 (header/main/footer/nav) ⚠️ 정적분석 Navbar.tsx:132 — Breadcrumbs가 <div>로 구현됨. 보조 내비게이션 역할을 하므로 <nav>가 적절함. Navbar 자체는 <nav> 적절히 사용 <nav aria-label="브레드크럼">로 변경
섹션 구조 (article/section/aside) ✅ 정적분석 네비게이션/필터 컴포넌트로 독립 콘텐츠 블록 없음
대화형 요소 (button/a vs div/span) ✅ 정적분석 모든 클릭 가능한 요소가 <button> 또는 <a>(Link) 사용. <div onClick> 패턴 없음
목록 구조 (ul/ol/li) ⚠️ 정적분석 Navbar.tsx:33-46 — 네비게이션 링크가 반복적으로 렌더링되지만 <ul><li> 구조 없이 <div> 내에 나열됨. SearchFilter.tsx:31-44 — 필터 버튼도 동일 네비게이션 링크를 <ul><li>로 감싸기: <ul role="list"><li><Link>...</Link></li></ul>
표현용 태그 (b/i 대신 strong/em) ➖ 정적분석 <b>, <i> 태그 사용 없음
폼 그룹 (fieldset/legend) ⚠️ 정적분석 SearchFilter.tsx:29-44 — 필터 버튼 그룹이 <div>로만 감싸져 있음. 필터 그룹의 용도를 스크린 리더에 전달하지 않음 <div role="group" aria-label="필터">로 변경하거나, <fieldset><legend>필터</legend>...</fieldset> 사용
기타 시맨틱 요소 (time/blockquote 등) ➖ 정적분석 날짜/인용 콘텐츠 없음

불합격 (❌)

항목 29. 레이블 제공 — SearchFilter.tsx:21-27

문제: <input type="search">에 <label>, aria-label, title 속성이 없음. placeholder="할 일 검색..."만 사용하고 있으나 placeholder는 레이블로 인정되지 않음 (KWCAG2.2 A-29)

수정: <input type="search" aria-label="할 일 검색" placeholder="할 일 검색..." ...>

권고 (⚠️)

항목 17. 반복 영역 건너뛰기

내용: Navbar.tsx에 스킵 내비게이션 링크가 없음. <a href="#main-content" className="skip-nav">본문 바로가기</a>를 <nav> 앞에 추가 권장

항목 23. 레이블과 네임 — Navbar.tsx:63

내용: 햄버거 버튼의 aria-label이 메뉴 상태에 따라 동적으로 변경되지 않음. aria-label={menuOpen ? "메뉴 닫기" : "메뉴 열기"}로 변경 권장

항목 33. 웹 애플리케이션 접근성

내용:

(1) Navbar.tsx:59 — 햄버거 버튼에 aria-expanded={menuOpen} 추가 필요. 메뉴 열림/닫힘 상태를 보조 기술에 전달해야 함

(2) SearchFilter.tsx:32 — 필터 버튼에 aria-pressed={filter === f.value} 추가 필요. 현재 선택된 필터를 보조 기술에 전달해야 함

시맨틱 HTML: 랜드마크 요소 — Navbar.tsx:132

내용: Breadcrumbs 컴포넌트가 <div>로 구현됨. 보조 내비게이션 역할이므로 <nav aria-label="브레드크럼">으로 변경 권장

시맨틱 HTML: 목록 구조

내용: 네비게이션 링크와 필터 버튼이 반복 렌더링되지만 <ul><li> 구조를 사용하지 않음. 스크린 리더가 항목 수를 안내할 수 있도록 리스트 마크업 사용 권장

시맨틱 HTML: 폼 그룹 — SearchFilter.tsx:29

내용: 필터 버튼 그룹에 role="group" aria-label="필터" 추가하여 그룹의 용도를 보조 기술에 전달 권장
