Navbar.tsx 검토 결과

1. 햄버거 메뉴 버튼 — aria-expanded 누락

Navbar.tsx · 59–72번 줄

불합격 KWCAG 2.2 — 4.1.2 이름, 역할, 값 (인식의 용이성)

모바일 햄버거 버튼에 aria-label="메뉴 열기"는 제공되어 있으나, 메뉴 열림/닫힘 상태를 나타내는 aria-expanded 속성이 없습니다. 스크린리더 사용자는 현재 메뉴가 열려 있는지 닫혀 있는지 알 수 없습니다.

수정 방법: aria-expanded={menuOpen}를 버튼에 추가하고, aria-label을 상태에 맞게 동적으로 변경하거나 aria-controls로 제어 대상을 명시합니다.

<button
  aria-label={menuOpen ? "메뉴 닫기" : "메뉴 열기"}
  aria-expanded={menuOpen}
  aria-controls="mobile-menu"
  ...
>

2. 모바일 메뉴 패널 — 역할 및 ID 누락

Navbar.tsx · 88번 줄

불합격 KWCAG 2.2 — 4.1.2 이름, 역할, 값

모바일 메뉴 컨테이너(<div>)에 id가 없어 햄버거 버튼의 aria-controls와 연결이 불가능합니다. 또한 메뉴가 동적으로 열릴 때 스크린리더에 변경이 알려지지 않습니다.

수정 방법: id="mobile-menu"를 추가하고, 필요 시 role="navigation" 또는 aria-label="모바일 메뉴"를 부여합니다.

3. 현재 페이지 링크 — aria-current 누락

Navbar.tsx · 34–45번 줄, 89–99번 줄

불합격 KWCAG 2.2 — 2.4.4 링크 목적 / 4.1.2 이름, 역할, 값

현재 경로(pathname === link.href)일 때 색상만 변경되고 aria-current="page"가 설정되지 않습니다. 스크린리더 사용자는 어떤 링크가 현재 페이지인지 알 수 없습니다.

수정 방법:

<Link
  ...
  aria-current={pathname === link.href ? "page" : undefined}
>

4. 브레드크럼브 — 시맨틱 마크업 및 aria-label 누락

Navbar.tsx · 131–147번 줄

불합격 KWCAG 2.2 — 2.4.8 위치, 1.3.1 정보와 관계

브레드크럼브가 <div>와 <span>으로만 구성되어 있으며, <nav>, <ol>/<li> 같은 시맨틱 요소가 사용되지 않았습니다. 또한 구분자 /가 장식용인데 스크린리더에 그대로 읽힙니다.

수정 방법:

<nav aria-label="브레드크럼브">
  <ol style="list-style:none;display:flex;...">
    <li><Link href="/">홈</Link></li>
    {segments.map((seg, i) => (
      <li key={seg}>
        <span aria-hidden="true"> / </span>
        {i === segments.length - 1
          ? <span aria-current="page">{labels[seg]}</span>
          : <Link href={...}>{labels[seg]}</Link>}
      </li>
    ))}
  </ol>
</nav>

5. 네비게이션 링크 목록 — <ul>/<li> 미사용

Navbar.tsx · 32–46번 줄

경고 KWCAG 2.2 — 1.3.1 정보와 관계

데스크톱 내비게이션 링크들이 <div> 안에 바로 나열되어 있습니다. 링크 목록은 <ul>/<li>로 묶어 목록 구조를 명시하는 것이 시맨틱 HTML 관례에 맞습니다.

수정 방법: 링크들을 <ul role="list">와 <li>로 감쌉니다. 단, CSS reset으로 기본 스타일을 제거해야 합니다.

6. SVG 아이콘 — 접근 가능한 텍스트 처리

Navbar.tsx · 65–71번 줄

경고 KWCAG 2.2 — 1.1.1 비텍스트 콘텐츠

SVG에 aria-hidden="true"가 없습니다. 버튼에 aria-label이 있어 최악의 상황은 피하지만, SVG 자체가 스크린리더에 의미 없는 내용으로 읽힐 수 있습니다.

수정 방법: <svg aria-hidden="true" focusable="false" ...>를 추가합니다.

7. 로그아웃 버튼 — 색상만으로 구분

Navbar.tsx · 48–54번 줄

합격 KWCAG 2.2 — 1.4.1 색상 사용

로그아웃 버튼은 빨간색으로 강조되어 있지만 텍스트("로그아웃")가 명확히 표시되어 있어 색상만으로 정보가 전달되지는 않습니다.

8. 주 내비게이션 <nav> 랜드마크

Navbar.tsx · 23번 줄

합격 KWCAG 2.2 — 2.4.1 블록 건너뛰기

<nav> 요소가 사용되어 랜드마크 역할을 수행합니다. 단, aria-label="주 내비게이션"을 추가하면 페이지에 여러 <nav>가 있을 때 구분에 도움이 됩니다.

SearchFilter.tsx 검토 결과

1. 검색 입력 필드 — <label> 누락

SearchFilter.tsx · 21–28번 줄

불합격 KWCAG 2.2 — 1.3.1 정보와 관계 / 3.3.2 레이블 또는 지시사항

<input type="search">에 placeholder만 있고 <label> 요소가 없습니다. placeholder는 레이블의 대체재가 아니며, 포커스 시 사라져 인지 부하를 증가시킵니다. 스크린리더가 입력의 목적을 정확히 전달하지 못합니다.

수정 방법:

<label htmlFor="todo-search" style={{display:"block",...}}>할 일 검색</label>
<input id="todo-search" type="search" ... />

/* 또는 시각적으로 숨기는 경우 */
<label htmlFor="todo-search" className="sr-only">할 일 검색</label>

2. 필터 버튼 — aria-pressed 누락

SearchFilter.tsx · 31–44번 줄

불합격 KWCAG 2.2 — 4.1.2 이름, 역할, 값

선택된 필터 버튼의 상태가 파란색 배경으로만 표시됩니다. 스크린리더 사용자는 어떤 필터가 현재 활성화되어 있는지 알 수 없습니다. 코드 주석에도 "BAD: missing aria-pressed"라고 명시되어 있습니다.

수정 방법:

<button
  aria-pressed={filter === f.value}
  onClick={() => onFilterChange(f.value)}
  ...
>
  {f.label}
</button>

또는 필터 그룹을 role="group"과 aria-label="필터"로 묶어 컨텍스트를 제공합니다.

3. 필터 버튼 그룹 — 그룹 역할 및 레이블 누락

SearchFilter.tsx · 29번 줄

불합격 KWCAG 2.2 — 1.3.1 정보와 관계

필터 버튼들을 감싸는 <div>에 그룹 역할이 없어, 이 버튼들이 관련 있는 항목들의 묶음이라는 것을 스크린리더가 파악하기 어렵습니다.

수정 방법:

<div role="group" aria-label="할 일 필터" ...>
  {filters.map(...)}
</div>

4. 전체 컨테이너 — 시맨틱 요소 부재

SearchFilter.tsx · 20번 줄

합격 KWCAG 2.2 — 1.3.1 정보와 관계

외부 <div>는 레이아웃 목적으로만 사용되어 큰 문제는 없지만, <search> (HTML5.3) 또는 <form role="search">로 감싸면 더 명확한 시맨틱을 제공할 수 있습니다.
