불합격 (❌)

항목 10. 키보드 사용 보장 — Card.tsx:24-29

문제: <div onClick={handleClick}> 비대화형 요소에 클릭 이벤트만 할당되어 있어 키보드(Tab/Enter/Space)로 접근 및 실행이 불가합니다. tabIndex, role, onKeyDown 핸들러가 모두 누락되어 있습니다.

수정:

방법 1 (권장): <div>를 <button>으로 변경

<button className={`rounded border p-4 ${className}`} onClick={handleClick}>

방법 2: ARIA 속성 추가

<div role="button" tabIndex={0} onClick={handleClick} onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') handleClick(); }}>

항목 32. 마크업 오류 방지 — Card.tsx:27

문제: id="card-item"이 하드코딩되어 있어, Card 컴포넌트가 리스트 등에서 여러 번 렌더링되면 동일한 id가 페이지 내에 중복 발생합니다. HTML 명세에 따르면 id는 문서 내에서 고유해야 합니다.

수정: id를 prop으로 전달받거나 고유 식별자를 생성합니다.

<div id={`card-${id}`} ...> 또는 id 속성 자체를 제거합니다.

시맨틱 HTML: 대화형 요소 — Card.tsx:24-29

문제: 클릭 가능한 <div onClick>가 네이티브 대화형 요소(<button>, <a>) 대신 사용되고 있습니다. 스크린 리더 사용자에게 이 요소가 클릭 가능하다는 정보가 전달되지 않습니다.

수정: <button> 요소로 변경하면 키보드 접근성, ARIA role, 포커스 표시가 자동으로 해결됩니다.

권고 (⚠️)

항목 11. 초점 이동과 표시

내용: 클릭 가능한 카드 영역에 tabIndex가 없어 키보드 초점이 이동하지 않으며, 초점 스타일도 정의되어 있지 않습니다. <button>으로 변경하면 자동 해결됩니다.

항목 33. 웹 애플리케이션 접근성

내용: 클릭 가능한 커스텀 <div> 컴포넌트에 WAI-ARIA role/속성/상태 정보가 없습니다. 네이티브 HTML 요소(<button>)로 변경하거나, 커스텀 요소 유지 시 role="button" 등 적절한 ARIA 속성을 추가하세요.

시맨틱 HTML: 섹션 구조

내용: Card 컴포넌트는 독립적인 콘텐츠 블록을 나타내므로 최상위 <div> 대신 <article> 요소 사용을 권고합니다.
