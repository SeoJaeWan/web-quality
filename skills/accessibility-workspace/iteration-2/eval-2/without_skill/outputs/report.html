# 심각도 KWCAG 2.2 기준 위치 문제 설명 개선 방안
1 심각 2.1.1 키보드 접근성 Line 24-30
<div onClick>		 <div> 요소에 onClick 핸들러가 있으나 tabIndex, onKeyDown/onKeyPress 이벤트가 없어 키보드로 해당 요소에 접근하거나 활성화할 수 없습니다. 키보드 사용자는 이 카드와 상호작용할 수 없습니다. tabIndex={0}을 추가하고, onKeyDown 핸들러에서 Enter/Space 키 입력 시 handleClick을 호출하도록 구현합니다. 또는 <button> 요소로 변경하는 것을 권장합니다.
2 심각 4.1.2 이름, 역할, 값 (Name, Role, Value) Line 24-30
<div onClick>		 클릭 가능한 <div> 요소에 role 속성이 없어 보조 기술(스크린 리더)이 이 요소가 상호작용 가능한 요소임을 인식할 수 없습니다. 사용자에게 클릭 가능한 영역임이 전달되지 않습니다. role="button" 또는 적절한 ARIA 역할을 추가합니다. 이상적으로는 <button> 또는 <a> 같은 네이티브 HTML 인터랙티브 요소를 사용하는 것이 바람직합니다.
3 심각 4.1.1 파싱 / 마크업 유효성 Line 26
id="card-item"		 id="card-item"이 하드코딩되어 있어 카드가 리스트로 렌더링될 경우 동일한 id가 여러 번 사용됩니다. HTML 명세상 id는 문서 내에서 고유해야 하며, 중복 id는 보조 기술의 탐색 기능을 방해합니다. id를 고유하게 생성하거나 (예: id={`card-${uniqueId}`}), 불필요한 경우 id 속성을 제거합니다.
4 주요 2.4.7 초점 표시 (Focus Visible) Line 24-30
<div>		 클릭 가능한 요소에 포커스 스타일이 정의되어 있지 않습니다. tabIndex를 추가하더라도 <div> 요소는 기본 포커스 표시가 브라우저에 따라 보이지 않을 수 있으며, 키보드 사용자가 현재 포커스 위치를 시각적으로 확인할 수 없습니다. CSS :focus-visible 또는 :focus 의사 클래스를 사용하여 명확한 포커스 표시 스타일(예: outline: 2px solid #005fcc)을 적용합니다.
5 주요 1.4.1 색의 사용 (Use of Color) Line 19
style.background = "yellow"		 클릭 시 배경색을 노란색으로 변경하여 상태를 전달하지만, 색상만으로 정보를 전달하고 있습니다. 색각 이상 사용자나 스크린 리더 사용자는 이 상태 변화를 인식할 수 없습니다. 색상 변경과 함께 텍스트, 아이콘, 또는 aria-pressed/aria-selected 같은 ARIA 상태를 추가하여 색상 외의 방법으로도 상태 변화를 전달합니다.
6 경미 1.3.1 정보와 관계 (Info and Relationships) Line 31
<h3>		 <h3>가 사용되었으나 상위 문맥에 <h1>, <h2>가 있는지 확인이 필요합니다. 카드 컴포넌트가 독립적으로 사용될 경우 제목 계층이 건너뛰어질 수 있으며, 이는 스크린 리더 사용자의 문서 구조 파악을 어렵게 합니다. 제목 수준을 prop으로 받아 동적으로 설정하거나 (예: as prop 패턴), 사용 문맥에 맞는 제목 수준을 사용하도록 문서화합니다.
