1.1.1 적절한 대체 텍스트 제공

dashboard/page.tsx — 83번째 줄

<img src="/promo-banner.png"> 요소에 alt 속성이 완전히 누락되어 있습니다. 스크린리더는 파일명을 읽거나 이미지를 무시해 사용자가 콘텐츠 목적을 파악할 수 없습니다.

{/* 의미 있는 이미지 */} <img src="/promo-banner.png" alt="신규 기능 홍보 배너" className="mt-6 w-full" /> {/* 순수 장식용 이미지 */} <img src="/promo-banner.png" alt="" role="presentation" className="mt-6 w-full" />
#2 아이콘 전용 버튼에 접근 가능한 이름 미제공 불만족 자동 판정 1. 인식의 용이성

1.1.1 적절한 대체 텍스트 제공

dashboard/page.tsx — 59~65번째 줄

새로고침·설정 버튼이 SVG 아이콘만으로 구성되며 aria-label도, SVG 내부 <title>도 없습니다. 스크린리더는 버튼 목적을 전혀 알릴 수 없습니다.

<button data-testid="btn-refresh"> <svg width="16" height="16" viewBox="0 0 16 16"> <circle cx="8" cy="8" r="8" /> </svg> </button> {/* aria-label, title 없음 → KWCAG 1.1.1 위반 */}

각 버튼에 aria-label을 추가하고 SVG에 aria-hidden="true"를 설정하십시오.

<button aria-label="새로고침" data-testid="btn-refresh"> <svg aria-hidden="true" focusable="false" width="16" height="16" viewBox="0 0 16 16"> <circle cx="8" cy="8" r="8" /> </svg> </button> <button aria-label="설정" data-testid="btn-settings"> <svg aria-hidden="true" focusable="false" width="16" height="16" viewBox="0 0 16 16"> <rect width="16" height="16" /> </svg> </button>
#3 활성 내비게이션 링크를 색상만으로 구분 불만족 수동 판정 1. 인식의 용이성

1.4.1 색에 무관한 콘텐츠 인식

Navbar.tsx — 37~42번째 줄

현재 페이지(활성) 내비게이션 링크는 파란색(text-blue-600)만으로 표시됩니다. 색각 이상 사용자는 현재 위치를 파악하기 어렵습니다. aria-current="page"도 없어 스크린리더 사용자도 현재 페이지 구분이 불가능합니다.

className={`... ${ pathname === link.href ? "text-blue-600 dark:text-blue-400" // 색상만으로 구분 → KWCAG 1.4.1 위반 : "text-zinc-600 hover:text-zinc-900 ..." }`}

색상 외에 밑줄, 굵기 등 비-색상 단서를 추가하고 aria-current="page"를 부여하십시오.

<Link href={link.href} aria-current={pathname === link.href ? "page" : undefined} className={`text-sm font-medium transition-colors ${ pathname === link.href ? "text-blue-600 dark:text-blue-400 font-bold underline decoration-2" : "text-zinc-600 hover:text-zinc-900 dark:text-zinc-400 dark:hover:text-zinc-50" }`} >
#4 이미지 크기 미지정으로 레이아웃 이동(CLS) 발생 주의 수동 판정 1. 인식의 용이성

1.3.4 콘텐츠의 재배치 (예기치 않은 이동 방지)

dashboard/page.tsx — 85번째 줄

/footer-ad.png에 width·height가 없어 로딩 중 레이아웃이 이동합니다. 인지 장애·저시력 사용자가 읽던 위치를 잃을 수 있습니다.

<img src="/footer-ad.png" alt="Advertisement" className="mt-4" /> {/* width, height 미지정 → CLS 발생, KWCAG 1.3.4 관련 */}

실제 이미지 비율에 맞는 width·height를 명시하거나 Next.js Image 컴포넌트를 사용하십시오.

import Image from "next/image"; <Image src="/footer-ad.png" alt="광고 배너" width={1200} height={200} loading="lazy" className="mt-4" />
#5 div 요소에 onClick — 키보드 접근 불가 불만족 자동 판정 2. 운용의 용이성

2.1.1 키보드 사용 보장

StatsCard.tsx — 10~22번째 줄

<div>에 onClick이 있으나 tabIndex, role, onKeyDown이 없습니다. 키보드 사용자는 Tab으로 포커스를 이동할 수 없고 Enter/Space로도 실행할 수 없습니다.

<div id="stat-card" onClick={handleClick} {/* tabIndex, role, onKeyDown 없음 → KWCAG 2.1.1 위반 */} >

인터랙티브 요소에는 <button>을 사용하는 것이 가장 간단하고 올바른 방법입니다.

{/* 권장: button 요소 사용 */} <button className="rounded-xl border border-zinc-200 bg-white p-6 text-left dark:border-zinc-800 dark:bg-zinc-900" onClick={handleClick} data-testid={testId} > <p className="text-3xl font-bold text-zinc-900 dark:text-zinc-50">{value}</p> <p className="mt-1 text-sm text-zinc-500">{label}</p> </button>
#6 중복 id 속성 사용 불만족 자동 판정 4. 견고성

4.1.1 마크업 오류 방지 (id 유일성)

StatsCard.tsx — 13번째 줄 (3개 인스턴스 렌더링)

StatsCard가 3개 렌더링될 때 모두 id="stat-card"를 갖습니다. HTML 스펙은 id가 문서 내 고유해야 함을 요구하며, 중복 id는 보조 기술이 잘못된 요소를 참조하게 하여 접근성 트리를 파손시킵니다.

<div id="stat-card" ...> {/* 3회 렌더링 → 중복 id → KWCAG 4.1.1 위반 */}

고유한 id가 필요하다면 props로 전달받거나 제거하십시오. data-testid는 이미 고유하게 설정되어 있으므로 id는 제거해도 무방합니다.

{/* id 제거 — data-testid로 충분 */} <div className="rounded-xl border ..." data-testid={testId} onClick={handleClick} > {/* 또는 고유 id 전달 */} export default function StatsCard({ label, value, testId, id }: {...}) { return <div id={id} ...>; }
#7 햄버거 메뉴 버튼 aria-expanded 미제공 불만족 자동 판정 2. 운용의 용이성

4.1.2 이름·역할·값 제공

Navbar.tsx — 59~72번째 줄

모바일 햄버거 버튼에 aria-label="메뉴 열기"는 있으나 aria-expanded가 없습니다. 스크린리더 사용자는 메뉴가 현재 열려 있는지 닫혀 있는지 알 수 없습니다. 또한 aria-label이 상태에 무관하게 "메뉴 열기"로 고정되어 있습니다.

<button aria-label="메뉴 열기" {/* 상태 미반영, aria-expanded 없음 */} onClick={() => setMenuOpen(!menuOpen)} >

aria-expanded를 상태에 따라 설정하고 aria-controls로 제어 대상을 연결하십시오.

<button aria-label={menuOpen ? "메뉴 닫기" : "메뉴 열기"} aria-expanded={menuOpen} aria-controls="mobile-nav-menu" onClick={() => setMenuOpen(!menuOpen)} > {menuOpen && isAuthenticated && ( <div id="mobile-nav-menu" ...> ... </div> )}
#8 반복 영역 건너뛰기 링크(Skip Navigation) 미제공 불만족 수동 판정 2. 운용의 용이성

2.4.1 반복 영역 건너뛰기

Navbar.tsx — 전체 (최상단 건너뛰기 링크 없음)

내비게이션 바 앞에 "본문 바로 가기" 링크가 없습니다. 키보드·스크린리더 사용자는 매 페이지 로드마다 내비게이션 전체를 탐색한 후에야 본문에 접근할 수 있습니다.

Navbar 최상단에 시각적으로 숨겨진 건너뛰기 링크를 추가하고, 메인 콘텐츠 래퍼에 id="main-content"를 부여하십시오.

{/* Navbar.tsx 반환 최상단 */} <a href="#main-content" className="sr-only focus:not-sr-only focus:absolute focus:top-4 focus:left-4 focus:z-50 focus:bg-white focus:px-4 focus:py-2 focus:rounded focus:shadow" > 본문 바로 가기 </a> {/* 레이아웃의 메인 영역 */} <main id="main-content" tabIndex={-1}> {children} </main>
#9 브레드크럼에 nav/aria-label 미적용 주의 수동 판정 3. 이해의 용이성

3.2.4 일관된 식별 / 2.4.8 현재 위치 정보 제공

Navbar.tsx — 131~146번째 줄 (Breadcrumbs 컴포넌트)

브레드크럼이 <div>로만 구성되어 스크린리더가 이를 내비게이션 랜드마크로 인식하지 못합니다. 현재 페이지 항목에 aria-current="page"도 없습니다. 구분자 "/" 텍스트를 스크린리더가 그대로 읽습니다.

<div className="..." data-testid="breadcrumbs"> {/* nav 랜드마크 없음, aria-label 없음, aria-current 없음 */} <Link href="/">홈</Link> {segments.map((seg, i) => ( <span key={seg}> <span className="mx-1">/</span> {/* 스크린리더가 "/" 읽음 */} ... </span> ))} </div>

<nav aria-label="현재 위치">와 <ol>/<li> 구조를 사용하고, 구분자는 aria-hidden="true"로 처리하며 현재 페이지에 aria-current="page"를 부여하십시오.

<nav aria-label="현재 위치" data-testid="breadcrumbs"> <ol style={{display:'flex', listStyle:'none', gap:'4px', fontSize:'14px'}}> <li><Link href="/">홈</Link></li> {segments.map((seg, i) => ( <li key={seg} style={{display:'flex', alignItems:'center', gap:'4px'}}> <span aria-hidden="true">/</span> {i === segments.length - 1 ? ( <span aria-current="page">{labels[seg] || seg}</span> ) : ( <Link href={...}>{labels[seg] || seg}</Link> )} </li> ))} </ol> </nav>
#10 nav 랜드마크에 aria-label 미제공 (복수의 nav 존재) 만족 (조건부) 수동 판정 2. 운용의 용이성

2.4.6 제목과 레이블 / 랜드마크 구분

Navbar.tsx — 23번째 줄

현재 <nav> 요소 하나이므로 현재 코드는 통과 기준을 충족하나, 브레드크럼 영역에 <nav>가 추가되면 두 <nav>를 구별하는 aria-label이 필요해집니다. 권고 사항으로 미리 추가할 것을 제안합니다.

<nav aria-label="주요 내비게이션" data-testid="navbar">
