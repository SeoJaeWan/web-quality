불합격 (❌)

항목 10. 키보드 사용 보장 (A-10) — StatsCard.tsx:10–21

문제: <div onClick={handleClick}>는 기본적으로 키보드 접근 불가. Tab 키로 포커스 이동 불가, Enter/Space 키 동작 없음. Playwright 확인: tabIndex=-1, 키보드 이벤트 핸들러 없음.

수정 (방법 1 — 권장):

<button
  type="button"
  id={testId}
  className="rounded-xl border border-zinc-200 bg-white p-6 ..."
  data-testid={testId}
  onClick={handleClick}
>

수정 (방법 2 — div 유지 시):

<div
  id={testId}
  role="button"
  tabIndex={0}
  aria-label={label}
  onClick={handleClick}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') handleClick();
  }}
  ...
>

항목 23. 레이블과 네임 (A-23) — StatsCard.tsx:10–21

문제: 클릭 가능한 div에 접근 가능한 이름(accessible name)이 없음. 스크린 리더 사용자가 이 요소의 목적을 알 수 없음.

수정: aria-label={`${label}: ${value}`} 추가 또는 button으로 교체 시 내부 텍스트가 이름으로 자동 사용됨.

항목 32. 마크업 오류 방지 (A-32) — StatsCard.tsx:13

문제: id="stat-card" 하드코딩으로 동일 페이지에서 여러 번 렌더링 시 동일 ID 중복 발생. Playwright 확인: DOM에 3개의 id="stat-card" 존재.

수정:

// 기존: id="stat-card" (하드코딩)
// 수정: id 속성 제거 (testId가 data-testid로 이미 제공됨)
<div
  className="rounded-xl ..."
  data-testid={testId}
  onClick={handleClick}
>

항목 33. 웹 애플리케이션 접근성 (A-33) — StatsCard.tsx:10–21

문제: 커스텀 인터랙티브 div에 WAI-ARIA role/속성 없음. 보조 기술이 이 요소를 일반 텍스트 블록으로 인식하여 상호작용 목적 파악 불가.

수정: button 교체(권장) 또는 role="button" + aria-label={label} 추가.

시맨틱 HTML: 대화형 요소 — StatsCard.tsx:10

문제: onClick이 있는 <div>는 시맨틱하지 않음. 클릭 가능한 요소는 <button> 또는 <a>를 사용해야 함.

수정: <div onClick>를 <button type="button">으로 교체. 이 하나의 수정으로 A-10, A-23, A-33, 시맨틱 HTML 문제가 동시에 해결됨.
