KWCAG 2.2 항목별 검토 결과

# 항목명 (KWCAG 2.2) 원칙 결과 판정방식 발견된 문제 수정 가이드
1 2.1.1 키보드 사용 보장
모든 기능은 키보드로 접근·조작 가능해야 함		 운용성 실패 코드 검토
<div onClick={handleClick}>에 tabIndex, role, onKeyDown/onKeyPress가 없어 키보드 사용자는 클릭 이벤트를 트리거할 수 없습니다. 마우스 없이는 이 기능에 접근 불가합니다.
<div>를 <button>으로 교체하거나, 불가피하게 <div>를 유지해야 할 경우 role="button", tabIndex={0}, onKeyDown 이벤트를 함께 추가하세요.
// 권장: button 요소 사용 <button type="button" onClick={handleClick}> ... </button>
2 1.1.1 적절한 대체 텍스트 제공
텍스트 아닌 콘텐츠에 대체 텍스트 제공		 인식성 실패 코드 검토
<p>{value}</p>는 숫자 값만 렌더링하며 해당 값이 무엇을 의미하는지 레이블이 없습니다. 스크린 리더 사용자는 숫자만 읽히고 맥락을 파악하기 어렵습니다. 카드 전체를 묶어주는 시맨틱 컨테이너나 aria-label/aria-labelledby도 없습니다.
카드 컨테이너에 aria-label 또는 aria-labelledby를 추가해 값과 레이블의 관계를 명시하거나, <dl>/<dt>/<dd> 구조를 사용해 의미를 구조화하세요.
<button aria-label={`${label}: ${value}`} ...> <p aria-hidden="true">{value}</p> <p aria-hidden="true">{label}</p> </button>
3 1.3.1 정보와 관계
콘텐츠 구조·관계는 프로그래밍 방식으로 표현되거나 텍스트로 이용 가능해야 함		 인식성 실패 코드 검토
카드가 단순 <div> 두 개의 <p>로 구성되어 있어 숫자(value)와 레이블(label) 간의 의미적 관계가 마크업에 표현되지 않습니다. 보조 기술은 두 텍스트가 서로 연관된 데이터(통계 수치와 그 설명)임을 인식할 수 없습니다.
시맨틱 마크업(<dl>, <figure>, <section> 등)을 사용하거나 ARIA 속성(role="group", aria-labelledby)으로 관계를 명시하세요.
<dl> <dt>{label}</dt> <dd>{value}</dd> </dl>
4 4.1.1 마크업 오류 방지
id 속성 값은 페이지 내 유일해야 함		 견고성 실패 코드 검토
id="stat-card"가 하드코딩되어 있어 동일 페이지에 StatsCard를 여러 번 렌더링하면 중복 id가 발생합니다. 중복 id는 HTML 명세 위반이며, ARIA 레이블링(aria-labelledby), 앵커 링크, 보조 기술의 탐색 등을 오동작하게 합니다.
id를 prop으로 받거나 useId()(React 18+)를 사용해 인스턴스별로 고유한 값을 생성하세요. 또는 id 속성 자체가 불필요하다면 제거하세요.
import { useId } from "react"; const uid = useId(); <div id={uid} ...>
5 2.4.7 포커스 가시성
키보드 조작 가능한 모든 UI 구성요소에 포커스 인디케이터가 표시되어야 함		 운용성 주의 코드 검토
현재 키보드 포커스 자체가 불가능하지만(항목 1 참조), 만약 button으로 수정하더라도 Tailwind CSS의 기본 스타일이 outline을 제거하는 경우 포커스 링이 보이지 않을 수 있습니다. 시각적 포커스 표시가 없으면 키보드 사용자가 현재 포커스 위치를 파악할 수 없습니다.
Tailwind의 focus-visible:ring-2 focus-visible:ring-blue-500 클래스를 추가하거나 CSS에서 :focus-visible 스타일을 명시적으로 정의하세요.
<button className="... focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500" ... >

수정된 코드 예시 (권장)

"use client"; import { useId } from "react"; interface StatsCardProps { label: string; value: number; testId: string; onClick?: () => void; } export default function StatsCard({ label, value, testId, onClick }: StatsCardProps) { const uid = useId(); const labelId = `${uid}-label`; const valueId = `${uid}-value`; return ( <button id={uid} type="button" onClick={onClick} data-testid={testId} aria-labelledby={`${valueId} ${labelId}`} className="rounded-xl border border-zinc-200 bg-white p-6 dark:border-zinc-800 dark:bg-zinc-900 text-left w-full focus-visible:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 focus-visible:ring-offset-2" > <p id={valueId} className="text-3xl font-bold text-zinc-900 dark:text-zinc-50">{value}</p> <p id={labelId} className="mt-1 text-sm text-zinc-500">{label}</p> </button> ); }

종합 의견

StatsCard 컴포넌트는 4개의 KWCAG 2.2 실패 항목과 1개의 주의 항목이 발견되었습니다. 가장 심각한 문제는 키보드 접근 불가로, 클릭 이벤트가 있는 <div>에 키보드 지원이 전혀 없어 키보드 사용자 및 스크린 리더 사용자가 이 기능을 전혀 이용할 수 없습니다. 또한 중복 id는 동일 페이지에서 복수 렌더링 시 보조 기술의 혼란을 야기합니다. 시맨틱 구조 부재로 인해 스크린 리더는 숫자 값과 레이블의 관계를 파악하지 못합니다.

근본적인 해결책은 <div>를 <button>으로 교체하고, useId()로 고유 ID를 생성하며, aria-labelledby로 값과 레이블의 관계를 명시하고, focus-visible 스타일을 적용하는 것입니다.
