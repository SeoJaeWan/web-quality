PERF Performance Issues (6)

High Layout thrashing in useEffect Lines 12-13

Reading offsetHeight (line 12) forces a synchronous reflow, then immediately writing style.minHeight (line 13) triggers a second reflow. This read-write cycle in a single frame causes layout thrashing and jank, especially on lower-powered devices.

Recommendation: Batch DOM reads and writes separately using requestAnimationFrame, or use ResizeObserver to react to size changes without forced reflows.
High Delayed content insertion causes CLS Lines 17-19

setTimeout injects three list items after 500ms without any placeholder or reserved space. This shifts existing content (including the promo image) downward, increasing Cumulative Layout Shift (CLS) -- a Core Web Vital that directly impacts user experience and search ranking.

Recommendation: Reserve space with a skeleton/placeholder of known dimensions, or load data before initial render to avoid layout shift.
Medium Image missing loading="lazy" Line 50

The <img> for promo.png is below the fold but lacks loading="lazy". The browser will fetch it eagerly even when not visible in the viewport, wasting bandwidth and potentially delaying Largest Contentful Paint (LCP).

Recommendation: Add loading="lazy" to below-fold images: <img loading="lazy" ... />.
Low Image missing explicit width and height Line 50

The <img> tag has no width/height attributes or aspect-ratio CSS. The browser cannot reserve space before the image loads, contributing to additional Cumulative Layout Shift.

Recommendation: Add explicit width and height attributes (e.g. width="800" height="400") or use CSS aspect-ratio.
Low List items rendered without stable keys Lines 45-47

items.map() does not provide a key prop. React falls back to array index, which causes unnecessary re-renders and DOM reconciliation overhead when the list changes.

Recommendation: Provide a stable, unique key: items.map((item) => <div key={item} ...>).
Low useEffect runs layout-affecting code unconditionally Lines 9-20

The useEffect with an empty dependency array runs layout-affecting code on every mount. Combined with setTimeout, this guarantees a visible layout shift on every page load.

Recommendation: Consider server-side or static data fetching (e.g. getServerSideProps / loader) to have data ready before first paint, eliminating both the forced reflow and the delayed content insertion.
