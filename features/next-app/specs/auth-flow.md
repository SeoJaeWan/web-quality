# 인증 흐름 테스트 플랜

**Base URL:** http://localhost:3000

---

### 1. 회원가입

**Seed:** `seed.spec.ts`

#### 1.1 유효한 회원가입
**Steps:**
1. /signup 페이지로 이동
2. 이름: "테스트유저" 입력
3. 이메일: "test@example.com" 입력
4. 비밀번호: "password123" 입력
5. 비밀번호 확인: "password123" 입력
6. "회원가입" 버튼 클릭

**Expected:** 회원가입 성공 후 /dashboard로 리다이렉트

#### 1.2 이메일 형식 오류
**Steps:**
1. /signup 페이지로 이동
2. 이메일: "invalid-email" 입력
3. 나머지 필드 유효하게 입력
4. "회원가입" 버튼 클릭

**Expected:** "올바른 이메일 형식이 아닙니다" 에러 메시지 표시

#### 1.3 비밀번호 불일치
**Steps:**
1. /signup 페이지로 이동
2. 비밀번호: "password123" 입력
3. 비밀번호 확인: "different456" 입력
4. "회원가입" 버튼 클릭

**Expected:** "비밀번호가 일치하지 않습니다" 에러 메시지 표시

#### 1.4 필수 필드 빈 값
**Steps:**
1. /signup 페이지로 이동
2. 모든 필드를 비운 채 "회원가입" 버튼 클릭

**Expected:** 각 필드에 대한 에러 메시지 표시

---

### 2. 로그인

**Seed:** `seed.spec.ts`

#### 2.1 유효한 로그인
**Steps:**
1. 먼저 회원가입으로 계정 생성
2. /login 페이지로 이동
3. 등록한 이메일과 비밀번호 입력
4. "로그인" 버튼 클릭

**Expected:** 로그인 성공 후 /dashboard로 리다이렉트

#### 2.2 미등록 이메일
**Steps:**
1. /login 페이지로 이동
2. 이메일: "notexist@example.com" 입력
3. 비밀번호: "password123" 입력
4. "로그인" 버튼 클릭

**Expected:** "등록되지 않은 이메일입니다" 에러 메시지 표시

#### 2.3 비밀번호 오류
**Steps:**
1. 먼저 회원가입으로 계정 생성
2. /login 페이지로 이동
3. 등록한 이메일 입력
4. 잘못된 비밀번호 입력
5. "로그인" 버튼 클릭

**Expected:** "비밀번호가 일치하지 않습니다" 에러 메시지 표시

### 3. 로그아웃

**Seed:** `seed.spec.ts`

#### 3.1 로그아웃
**Steps:**
1. 로그인 상태에서 "로그아웃" 버튼 클릭
