# 네비게이션 테스트 플랜

**Base URL:** http://localhost:3000
**전제조건:** 로그인 완료 상태

---

### 1. 클라이언트 사이드 라우팅

**Seed:** `seed.spec.ts`

#### 1.1 네비게이션 바 링크 전환
**Steps:**
1. /dashboard 페이지에서 시작
2. "할 일" 네비게이션 링크 클릭
3. URL이 /todos로 변경되는지 확인
4. "프로필" 네비게이션 링크 클릭
5. URL이 /profile로 변경되는지 확인

**Expected:** 페이지 전체 새로고침 없이 URL이 변경되고 해당 페이지 콘텐츠 표시

#### 1.2 브라우저 뒤로가기
**Steps:**
1. /dashboard → /todos → /profile 순서로 이동
2. 브라우저 뒤로가기 버튼 클릭
3. /todos 페이지인지 확인
4. 다시 뒤로가기 클릭
5. /dashboard 페이지인지 확인

**Expected:** 히스토리 스택에 따라 올바르게 뒤로 이동

#### 1.3 직접 URL 접근 (딥링크)
**Steps:**
1. 브라우저에서 직접 http://localhost:3000/todos 입력
2. 페이지가 정상 렌더링되는지 확인

**Expected:** /todos 페이지가 정상적으로 표시됨

---

### 2. 반응형 네비게이션

**Seed:** `seed.spec.ts`

#### 2.1 모바일 햄버거 메뉴
**Steps:**
1. 뷰포트를 모바일 크기(375px)로 설정
2. 햄버거 메뉴 버튼 클릭
3. 모바일 메뉴 표시 확인
4. "할 일" 링크 클릭
5. 메뉴 닫힘 및 페이지 이동 확인

**Expected:** 햄버거 메뉴가 열리고, 링크 클릭 시 메뉴 닫히며 페이지 이동

---

### 3. 브레드크럼

**Seed:** `seed.spec.ts`

#### 3.1 브레드크럼 표시
**Steps:**
1. /dashboard 페이지로 이동
2. 브레드크럼에 "홈 / 대시보드" 표시 확인
3. /todos 페이지로 이동
4. 브레드크럼에 "홈 / 할 일" 표시 확인

**Expected:** 현재 위치에 맞는 브레드크럼 경로 표시
