# 할 일 관리 테스트 플랜

**Base URL:** http://localhost:3000
**전제조건:** 테스트 실행 전 회원가입/로그인 완료 상태

---

### 1. 할 일 추가

**Seed:** `seed.spec.ts`

#### 1.1 유효한 할 일 추가
**Steps:**
1. /todos 페이지로 이동
2. 할 일 입력란에 "테스트 할 일 항목" 입력
3. "추가" 버튼 클릭
4. 할 일 목록에 "테스트 할 일 항목"이 표시되는지 확인

**Expected:** 할 일이 목록에 추가되고 입력란이 비워짐

#### 1.2 빈 입력 유효성 검증
**Steps:**
1. /todos 페이지로 이동
2. 입력란을 비운 채 "추가" 버튼 클릭
3. 에러 메시지 확인

**Expected:** "할 일을 입력해주세요" 에러 메시지 표시

#### 1.3 최대 길이 초과
**Steps:**
1. /todos 페이지로 이동
2. 200자 초과 텍스트 입력
3. "추가" 버튼 클릭
4. 에러 메시지 확인

**Expected:** "할 일은 200자 이하여야 합니다" 에러 메시지 표시

---

### 2. 할 일 조회 및 필터

**Seed:** `seed.spec.ts`

#### 2.1 전체 목록 조회
**Steps:**
1. 할 일 여러 개 추가
2. "전체" 필터 선택
3. 모든 할 일이 표시되는지 확인

**Expected:** 추가한 모든 할 일이 목록에 표시됨

#### 2.2 검색으로 필터링
**Steps:**
1. 할 일 여러 개 추가 (예: "장보기", "운동하기", "독서")
2. 검색란에 "장보기" 입력
3. 검색 결과 확인

**Expected:** "장보기"만 목록에 표시됨

#### 2.3 검색 결과 없음
**Steps:**
1. 검색란에 "존재하지않는항목" 입력
2. 빈 상태 UI 확인

**Expected:** "검색 결과가 없습니다" 메시지 표시

---

### 3. 할 일 수정

**Seed:** `seed.spec.ts`

#### 3.1 할 일 텍스트 수정
**Steps:**
1. 할 일 추가
2. "수정" 버튼 클릭
3. 텍스트를 "수정된 할 일"로 변경
4. "저장" 버튼 클릭
5. 변경된 텍스트 확인

**Expected:** 할 일 텍스트가 "수정된 할 일"로 업데이트됨

#### 3.2 할 일 완료 토글
**Steps:**
1. 할 일 추가
2. 체크박스 클릭
3. 완료 상태 확인 (취소선)

**Expected:** 할 일에 취소선이 표시되고 완료 상태로 변경됨

---

### 4. 할 일 삭제

**Seed:** `seed.spec.ts`

#### 4.1 할 일 삭제
**Steps:**
1. 할 일 추가
2. "삭제" 버튼 클릭
3. 확인 다이얼로그에서 "예" 클릭
4. 목록에서 삭제 확인

**Expected:** 할 일이 목록에서 제거됨

#### 4.2 삭제 취소
**Steps:**
1. 할 일 추가
2. "삭제" 버튼 클릭
3. 확인 다이얼로그에서 "아니오" 클릭
4. 할 일이 여전히 목록에 있는지 확인

**Expected:** 할 일이 삭제되지 않고 목록에 유지됨

---

### 5. 빈 상태

**Seed:** `seed.spec.ts`

#### 5.1 할 일 없을 때 빈 상태 UI
**Steps:**
1. 모든 할 일 삭제 (또는 새 계정으로 접속)
2. /todos 페이지 접속
3. 빈 상태 메시지 확인

**Expected:** "아직 할 일이 없습니다" 메시지와 안내 텍스트 표시
